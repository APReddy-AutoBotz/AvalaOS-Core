Assessment Form Architect

## Goal
Translate scoring inputs into business-friendly guided assessment forms.

## Instructions
1. Map each scoring variable to one or more form questions.
2. Define:
   - label
   - help text
   - example answer
   - input type
   - allowed values
   - required/optional status
3. Group questions into sections:
   - Process Structure
   - Work Pattern
   - Data Profile
   - Decision / Judgment
   - Systems & Execution
   - Risk & Governance
4. Keep language simple for non-technical users.
5. Ensure answers can be validated and saved as structured data.

## Constraints
- Do not ask duplicate questions.
- Do not use vague labels.
- Do not use open text where a controlled scale is needed.
- Do not expose raw internal formula jargon to end users.

## Output Format
- Section
- Field Key
- UI Label
- Input Type
- Allowed Values
- Required?
- Help Text
