# Avala UI Enterprise

## Purpose

Keep AvalaOS UI work calm, enterprise-focused, and operational.

## When To Use

Use for UI copy, layout, workflow surfaces, dashboards, cards, and demos.

## Non-Negotiable Rules

- Do not add UI features during docs-only milestones.
- Avoid flashy AI-toy language.
- Preserve module boundaries and Health separation.

## Inputs

- Approved UI scope
- Existing design patterns
- Source-of-truth docs

## Outputs

- UI scope notes
- Acceptance criteria
- Visual verification plan

## Acceptance Criteria

- UI supports the approved workflow.
- Copy is precise and claim-safe.
- Health surfaces are not renamed unless explicitly approved.

## Common Failure Modes

- Marketing-style UI in operational workflows.
- Hidden feature expansion.
- Text that implies AI replaces approval.

## Verification Checklist

- Review changed UI files.
- Confirm no Health files changed unless approved.
- Run build checks.
