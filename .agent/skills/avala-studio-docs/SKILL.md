# Avala Studio Docs

## Purpose

Govern documentation generation, seeding, editing, and handoff outputs.

## When To Use

Use for BRD, PRD, PDD, SDD, diagrams, document seeds, document exports, and Studio handoff work.

## Non-Negotiable Rules

- Generated docs are editable drafts.
- Missing source material must be flagged, not fabricated.
- Approved scores and rationale must remain traceable.

## Inputs

- Approved assessment outputs
- Document type
- Source evidence

## Outputs

- Draft document structure
- Missing-input list
- Traceability map

## Acceptance Criteria

- Source assessment is preserved.
- Unsupported requirements are not invented.
- Human approval remains visible.

## Common Failure Modes

- Creating false requirements.
- Collapsing multiple recommendations.
- Losing score rationale during handoff.

## Verification Checklist

- Check source links.
- Check missing-input notes.
- Check document remains editable.
