# Rule: Core Feature Lockdown
> Priority: CRITICAL

Strictly follow these constraints for all agentic tasks. `docs/00_SOURCE_OF_TRUTH.md` is the controlling product and architecture reference.

### 1. Read-Only Protection
The following files are LATCHED. You may READ them to understand context, but you are FORBIDDEN from writing to or modifying them without explicit, one-time permission:
- `services/geminiProvider.ts` (AI provider boundary)
- `services/prompts.ts` (prompt boundary)
- `components/widgets/*.tsx` (Dashboard UI)
- `types.ts` & `data/docTemplates.ts` (Data Structure)

### 2. Mandatory Approval Workflow
- Before broad file modification, generate an implementation plan or evidence checklist.
- Wait for explicit user approval when the task is in planning mode or when the requested change is ambiguous.

### 3. Checkpointing
- For every approved change, you MUST perform a `git commit` with a clear message describing the change.
- This creates a persistent "undo stack" for the developer.
