# Compliance Claim Boundary

Reference: `docs/00_SOURCE_OF_TRUTH.md`.

## Rule

AvalaOS Core may describe implemented controls, evidence records, approvals, audit events, and security boundaries. It must not claim formal regulatory or certification status unless that status has been formally validated and approved.

## Allowed Language

- "Designed to support evidence collection."
- "Records approval and audit evidence."
- "Requires formal validation before compliance claims."

## Prohibited Language

- Claims that AvalaOS Core is certified, compliant, validated, or approved for a regulated framework without formal support.
- Claims that a demo or prototype satisfies regulatory obligations.

## Review

Any compliance-related wording must pass source-of-truth review, legal/product approval, and evidence review before release.
