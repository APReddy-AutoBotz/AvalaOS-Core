# Incident And Kill Switch Model

Reference: `docs/00_SOURCE_OF_TRUTH.md`.

## Incident Triggers

- Unauthorized action.
- Action outside allowed boundaries.
- Missing approval for required risk level.
- Sensitive data exposure.
- Evidence or audit failure.
- Provider or integration malfunction.

## Kill Switch Requirements

- Owner identified.
- Stop condition defined.
- Affected process and systems recorded.
- Audit event created.
- Recovery decision documented.

## M0.2 Boundary

This document defines the model only. It does not add runtime controls.
