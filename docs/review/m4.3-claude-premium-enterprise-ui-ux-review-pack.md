# M4.3 Claude Premium Enterprise UI/UX Review Pack

## Reviewer Role

You are reviewing AvalaOS Core as a premium enterprise UI/UX reviewer.

Be direct and critical. Separate true buyer-demo blockers from polish, enterprise-hardening items, and ideas that can wait.

## Product Context

AvalaOS Core is a governed AI and automation delivery operating system.

It is not:

- a generic project management tool;
- a Jira clone;
- an AI chatbot;
- an autonomous agent builder;
- a simple document generator;
- a process mining tool;
- a side project or toy demo.

Core thesis:

- Evaluate before you automate.
- Govern before you execute.
- Humans approve risk.
- Evidence proves every decision.

Buyer-facing lifecycle:

`Avala Assess -> Avala Govern -> Avala Studio -> Avala Delivery -> Avala Monitor`

Brand meaning:

`AvalaOS = Assess. Validate. Align. Launch. Audit.`

## Current Golden Path

Review the current buyer-demo path:

`Avala Demo Enterprise -> AP Invoice Exception Handling / AP Invoice Exception Workflow -> Avala Assess -> Avala Govern -> Avala Studio -> Avala Delivery -> Avala Monitor`

Key interpretation:

- Avala Assess evaluates the AP process and produces deterministic scoring.
- Avala Govern is the control-card context: Decision Pack, human review, evidence, assumptions, and governed handoff.
- Avala Studio presents generated review artifacts from source context.
- Avala Delivery presents traceable work items and Delivery Pack readiness.
- Avala Monitor presents demo project/task/readiness signals, not live production telemetry.

## Screenshots / Manual Review Placeholder

Use screenshots or a live walkthrough from a clean browser session if available.

Minimum screens to review:

1. Login / first impression.
2. Module lifecycle strip.
3. Avala Assess process catalog and process detail.
4. Decision Pack and Avala Govern card.
5. Avala Studio handoff/source context and generated artifacts.
6. Avala Delivery work items.
7. Delivery Pack.
8. Avala Monitor project/task/readiness signals.

If screenshots are incomplete, note what cannot be assessed visually.

## Focus Areas

Review:

- first impression of landing/login screen;
- whether AvalaOS looks premium and enterprise-grade;
- visual hierarchy and layout clarity;
- whether module names feel professional;
- whether Assess -> Govern -> Studio -> Delivery -> Monitor is easy to understand;
- whether the AP Invoice Exception demo flow is easy to follow;
- confusing labels, weak CTAs, empty states, disabled areas;
- whether "Evaluate. Govern. Deliver. Prove." is clear;
- whether "Assess. Validate. Align. Launch. Audit." is clear;
- whether anything looks unfinished, demo-like, or low-trust;
- whether buyer-facing removal of "Lite" improves perception;
- what must be fixed before buyer demo;
- what can wait.

## Do Not Focus On

Do not:

- rewrite AvalaOS into a generic project management tool;
- rewrite AvalaOS into a chatbot;
- rewrite AvalaOS into a process mining tool;
- suggest broad new features unless they are true demo blockers;
- critique missing enterprise backend maturity as a UI blocker unless it visibly harms the buyer demo;
- assume live provider execution or production telemetry is available.

## Safety Review Instructions

Flag any copy that appears to claim:

- certified compliance;
- autonomous approval;
- AI-owned scoring, gates, risk, or recommendations;
- live production monitoring;
- live provider execution;
- full enterprise-production deployment.

## Output Format

Return:

1. Overall UI/UX verdict.
2. Premium enterprise impression score from 1-10.
3. Top 5 strengths.
4. Top 5 UI/UX concerns.
5. Confusing labels or weak copy.
6. Buyer-trust risks.
7. Demo blockers.
8. Fix recommendations ranked:
   - Must fix before buyer demo.
   - Acceptable with talk track.
   - Defer to enterprise hardening.
   - Ignore / not relevant.

Keep the critique practical. If something is only an enterprise-hardening issue, say so.
