# M4.2g Buyer-Demo Smoke Test And UI Readiness Checklist

## Purpose

This checklist supports a buyer-demo smoke test for the current canonical AP demo path:

`Avala Demo Enterprise -> AP Invoice Exception Handling / AP Invoice Exception Workflow`

It is a validation checklist only. It does not add features, change UI, change routing, change data, call providers, deploy, or implement reset behavior.

## Expected Buyer-Safe Wording

Use these phrases:

- "Evaluate before you automate."
- "Govern before you execute."
- "Deterministic scoring."
- "Decision Pack."
- "Control card."
- "Human review."
- "Evidence and assumptions."
- "Governed handoff."
- "Generated review artifacts."
- "Readiness and blocker signals."

## No-Go Claims

Do not claim:

- formal regulatory certification;
- model-owned approvals, scores, risk gates, or recommendations;
- live provider execution when providers are not deliberately enabled;
- live production monitoring;
- external system posting;
- vendor communication;
- final risk approval without human review;
- export enforcement before the deferred export-policy milestone.

## Known Limitations

- Visual browser validation must be executed by AP or the demo operator if local browser tooling is not available in the verification environment.
- Browser persisted state can mask canonical defaults.
- Brand meaning copy is not explicit in the current app and needs AP decision before implementation.
- Govern Lite remains a lightweight control-card context, not full Avala Govern.
- Monitor Insights remain partial/disabled beyond current project/task signals.
- Template-library copy remains outside this checklist and needs separate AP approval if changed.
- Reports, Teams, Admin/Workspace behavior, Admin AI Controls, and export-policy enforcement remain deferred or AP-decisioned.
