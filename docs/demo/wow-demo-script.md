# Wow Demo Script

Reference: `docs/00_SOURCE_OF_TRUTH.md`.

## Opening

"AvalaOS helps a team decide what should be automated, what must stay human-approved, and what evidence proves the decision."

For this walkthrough, use the prepared Month-End Close Evidence Pack so the demo focuses on the governed path, not live generation setup.

## Sequence

1. Show Month-End Close process intake and evidence capture.
2. Show deterministic assessment output and recommendation rationale.
3. Show the operating model and rejected alternatives.
4. Open the Avala Govern Lite card and highlight owner, autonomy, risk, approval, evidence, allowed actions, and blocked actions.
5. Show prepared Avala Studio documents from approved assessment data. Do not rely on live generation for the standard external demo.
6. Open the Delivery Pack and show source lineage, approval/evidence checklists, blockers, work items, and Markdown/JSON exports.
7. Show Monitor as an evidence-backed portfolio view for value, risk, blockers, and handoff status.

## Presenter Notes

- Start with deterministic assessment and evidence so the audience sees that scoring and recommendations are not AI-decided.
- Treat Govern Lite as the control moment: humans approve risk before handoff.
- Use prepared Studio documents for the default demo. Live Studio generation is optional only when approved provider controls are configured.
- In Delivery Pack, point to lineage, checklist status, blockers, and local exports.
- In Monitor, close with portfolio visibility into value, risk, blockers, and evidence-backed handoff.

## Close

"The point is not to make AI act faster. The point is to make governed AI and automation delivery provable."
