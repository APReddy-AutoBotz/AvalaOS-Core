# Wow Demo Script

Reference: `docs/00_SOURCE_OF_TRUTH.md`.

## Opening

"AvalaOS helps a team decide what should be automated, what must stay human-approved, and what evidence proves the decision."

For this walkthrough, use the prepared AP Invoice Exception Handling story so the demo focuses on the governed path, not live generation setup.

## Sequence

1. Show AP Invoice Exception Handling process intake and evidence capture.
2. Show deterministic assessment output and recommendation rationale.
3. Show the Decision Pack rationale, controls, evidence, assumptions, and handoff readiness.
4. Open the Avala Govern Lite card and highlight owner, autonomy, risk, approval, evidence, allowed actions, and blocked actions.
5. Show prepared Avala Studio review artifacts from approved source context. Do not rely on live generation for the standard external demo.
6. Open the AP Delivery Pack and show source lineage, approval/evidence checklists, blockers, and work items.
7. Show Monitor as an evidence-backed AP readiness view for value, risk, blockers, and handoff status.

## Presenter Notes

- Start with deterministic assessment and evidence so the audience sees that scoring and recommendations are not AI-decided.
- Treat Govern Lite as the control moment: humans approve risk before handoff.
- Use prepared Studio review artifacts for the default demo. Live Studio generation is optional only when approved provider controls are configured.
- In Delivery Pack, point to lineage, checklist status, blockers, and review output.
- In Monitor, close with AP readiness visibility into value, risk, blockers, and evidence-backed handoff.

## Close

"The point is not to make AI act faster. The point is to make governed AI and automation delivery provable."
