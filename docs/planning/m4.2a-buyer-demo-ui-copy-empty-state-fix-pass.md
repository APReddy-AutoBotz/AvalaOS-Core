# M4.2a Buyer-Demo UI Copy And Empty-State Fix Pass

## Purpose

M4.2a applies the focused UI copy and empty-state fix pass recommended by M4.2 before canonical demo data is implemented.

The goal is to make the visible app flow align with the intended buyer-demo story:

`Evaluate before you automate -> Assess -> Govern Lite -> Studio -> Delivery -> Monitor`

This milestone corrects stale labels, misleading CTAs, disabled-state copy, and empty-state wording without redesigning screens or changing product behavior.

## Baseline

- Prior milestone: `M4.2 UI Reality Check And Golden Path Screen Audit`.
- Prior tag: `avalaos-core-m4.2-ui-reality-check-golden-path-screen-audit`.
- Accepted M4.2 outcome: core golden-path screens exist, but the UI is not buyer-demo clean until stale labels, AI-first wording, export/lineage presentation, and empty states are tightened.

## Strict Scope

Allowed:

- Buyer-demo UI copy fixes.
- Empty-state wording fixes.
- Disabled-state wording fixes.
- Small CTA label and primary-action priority adjustments needed to support `Evaluate before you automate`.
- The two M4.2a planning/evidence docs.

Not allowed:

- Product redesign.
- Mock/demo data changes.
- Routing behavior changes.
- Deterministic scoring changes.
- Recommendation, gate, risk, or threshold changes.
- Schema, RLS, migration, or Supabase Edge Function changes.
- Runtime/provider behavior changes.
- Health changes.
- Canonical demo data implementation.
- M4.1g export block/label policy implementation.
- Admin/AI Controls route model implementation.
- Reports/Teams final behavior implementation.
- Dependency changes.
- Deployment or provider calls.
- Secrets access or printing.

## Implementation Summary

### App Shell And Golden Path Start

- The Header primary action now prefers Avala Assess before Avala Studio when both modules are available.
- The primary action copy now says `Start with process evaluation` for Assess.
- The App-shell primary candidate order now starts with Process Catalog before Studio.

This is a CTA priority and copy alignment change only. It does not change the central guard, Sidebar metadata, ModuleJourney metadata, or module enablement rules.

### Assess And Govern Lite Flow

- Guided Assessment scoring CTAs now say `Calculate deterministic score`.
- The side-panel Docs and Delivery status actions now record handoff snapshots instead of implying that documents or delivery backlog are generated in that step.
- The handoff readiness preview now explains that deterministic scoring records a Docs and Delivery handoff pack for human review.
- Process Detail Studio handoff now says `Open Avala Studio handoff`.

### Studio And Generated Documents

- Docs repository empty state now points users to Avala Studio, not an AI Workspace.
- Project selector copy now says Avala Studio drafts documents in a selected project context.
- Studio setup copy now uses `Avala Studio Controls` and `Draft provider`.
- Studio generation error copy is buyer-safe and does not expose raw provider details to the user.
- Generated Artifacts now labels export output as review artifacts and reminds users to confirm lineage, evidence refs, and approvals before sharing.
- The section refinement affordance now says `Refine Draft`.

### Docs-To-Delivery And Delivery Pack

- Import modal now states that imported items retain available lineage and evidence refs.
- Import modal now states that docs-only lineage remains partial and is not Assess-backed evidence.
- Generated work items panel now reminds users to treat partial lineage as review-needed until the export policy milestone is implemented.
- Delivery Pack export area now labels exports as review artifacts and asks users to review incomplete lineage or missing evidence refs before external sharing.

This is labeling only. M4.1g export block/label enforcement remains deferred.

### Monitor And Deferred Views

- Dashboard widget label changed from `AI Insights` to `Monitor Insights`.
- The disabled Monitor insight state now explains that Monitor insights are not enabled in the workspace while the rest of the demo flow remains available.
- Reports placeholder copy now points users to Avala Monitor and Delivery Pack for the demo-ready evidence flow.

### Setup And Template Library

- Organization setup removed visible `Upgrade to Premium` and `Coming Soon` wording.
- Starter pack UI now displays `Workspace Pack` and masks legacy premium wording in the visible pack description.
- Locked template action copy now says `Requires workspace configuration`.

The starter-pack source data was not changed.

## Acceptance Criteria

- Visible golden-path copy starts with assessment and deterministic scoring.
- Empty states use Avala product language, not stale workspace or assistant naming.
- Disabled/deferred surfaces explain what is available without looking broken.
- Export/import copy warns that lineage and evidence refs must be reviewed.
- M4.1g enforcement remains deferred.
- No product code behavior, mock/demo data, scoring, schema, provider runtime, Health, or dependency changes occur.
