## Root Cause Hypothesis

M3.2w evidence showed remote Supabase bundling failed while resolving `_shared/providerResolver` referenced from `_shared/providerResolverIntegration.ts`.

The M3.2x import-graph inspection found the resolver shared-module chain used extensionless sibling imports in deployed Edge modules, including:

- `providerResolverIntegration.ts` importing `./providerResolver`.
- `providerResolver.ts` importing `./providerResolverAudit`.
- Resolver DB/audit/secret adapter modules importing sibling shared modules without `.ts`.

Node-oriented local test emit tolerated those imports, but Supabase Edge remote bundling follows Deno-style module resolution and requires explicit local file extensions.
