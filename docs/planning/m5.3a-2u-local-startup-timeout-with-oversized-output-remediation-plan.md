## Recommended Remediation Path

Recommended next implementation: harden the classifier output-budget handling and timeout/oversized-output stop behavior.

Future M5.3a-2v should:

- Preserve the existing classifier interface unless AP separately approves an interface change.
- Define explicit redacted output-budget thresholds for output size and line count.
- Stop processing before unbounded output growth creates unsafe evidence pressure.
- Emit only the existing sanitized category and redacted signal fields unless AP approves new redacted fields.
- Keep `local-startup-timeout-with-oversized-output` as a signal category, not a root-cause claim.
- Preserve `unknown-local-startup-failure` as the safe fallback.
- Preserve scratch cleanup and fail closed if cleanup cannot be verified.
- Preserve no-raw-log, no-local-path, and no-secrets confirmations.

## Safety Controls

Future implementation must:

- Emit only labels, enum values, counts, buckets, booleans, pass/fail status, and sanitized classifications.
- Never print raw output, raw logs, local paths, target values, host values, port values, DB URLs, local config values, container IDs, image IDs, `.env` values, tokens, provider keys, row payloads, auth headers, or claim values.
- Fail closed if output safety validation fails.
- Fail closed if scratch cleanup fails.
- Fail closed if redacted signal generation cannot complete without copying raw or local values.
- Avoid local environment inference from low-confidence redacted signals.
- Keep evidence phrasing clear that remediation improves bounded handling, not local startup success.

## Stop Conditions

Future implementation must stop and report a safe blocked or fail-closed result if:

- Output-budget thresholds are exceeded before a safe redacted result can be emitted.
- The implementation would need raw output to decide classification.
- The implementation would need local path, target, config, environment, container, image, or log details.
- Any safety confirmation fails.
- Scratch cleanup fails.
- The result would imply root cause from redacted timeout and oversized-output signals.
- The result would claim local DB availability, schema readiness, artifact isolation, tenant isolation, hosted readiness, or production readiness.
