# M4.1a Decision Pack Contract And Rendering Fix

## Purpose

M4.1a fixes the first accepted M4.1 implementation milestone: scored Process Detail must render the buyer-facing Decision Pack safely from the current deterministic `DecisionPack` and `HandoffPack` contracts.

This milestone addresses M4.0a finding `M4A-C01` only. It does not implement downstream handoff payload, lineage, route guard, demo mode, Admin, provider, schema, or Health work.

## Scope

- Diagnose the Process Detail Decision Pack rendering contract mismatch.
- Keep deterministic scoring outputs unchanged.

## Root Cause

`components/assess/ProcessDetailStubView.tsx` rendered stale flat fields:

- `scores.decisionPack?.whyThisRecommendation`
- `scores.decisionPack?.governanceControls`

The current contract exposes:

- `scores.decisionPack.recommendedOperatingModel.whyThis`
- `scores.decisionPack.governance.auditControls`
- `scores.decisionPack.governance.dataControls`
- `scores.decisionPack.governance.monitoringControls`
- `scores.decisionPack.governance.securityControls`
- `scores.decisionPack.governance.modelProviderControls`
- `scores.handoffPack.governanceControls`

The mismatch could prevent scored Process Detail from rendering the Decision Pack rationale and governance proof point reliably.

## Implementation Plan

1. Add a small render-model adapter for Decision Pack display data.
2. Update Process Detail to read from the adapter instead of stale flat fields.
3. Add empty-state fallbacks for missing rationale, controls, document types, and backlog seeds.
4. Add a focused TypeScript regression for current-contract records, handoff-control fallback, and stale/partial score records.
5. Wire the focused regression into `npm run test`.

## Acceptance Criteria

- Scored Process Detail does not reference non-contract Decision Pack fields.
- Decision rationale is sourced from `recommendedOperatingModel.whyThis`.
- Governance controls are sourced from current nested governance control groups.
- Partial or older score records do not crash rendering data normalization.
- Deterministic scoring regressions remain unchanged and passing.
- No scoring, provider, schema, migration, Health, route guard, or downstream handoff changes are introduced.
