# M5.3a-2 Artifact SELECT Policy Implementation Plan

## Purpose

M5.3a-2 plans SELECT-only RLS policies for the applied artifact and authority tables that depend on the closed M5.3a-1 identity, organization, workspace, role, and membership helper foundation.

This milestone is docs-only. It does not implement artifact RLS policies, SQL migrations, helper changes, assertion SQL, harness changes, runtime adapters, UI behavior, package changes, deployment changes, database commands, remote validation, or provider calls.

## Accepted Baseline

M5.3a-1 is closed with post-merge verification and tag target `5fd3a29e6f5b803e73887e6df01a7bd0dcb6e1c7`.

Accepted foundation:

- `public.is_active_org_member(uuid)` exists.
- `public.is_active_workspace_member(uuid, uuid)` exists.
- Helpers are `SECURITY DEFINER`, `STABLE`, boolean-only, schema-qualified, and hardened with `search_path = pg_catalog`.
- Helper execute is revoked from `PUBLIC` and `anon`, and granted only to `authenticated`.
- SELECT-only policies exist for `profiles`, `organizations`, `workspaces`, `roles`, `organization_members`, and `workspace_memberships`.
- The roles policy explicitly requires `is_system = false`.
- No artifact policies exist yet.
- Tenant isolation is not verified because no reachable AP-approved local DB ran executable assertions.

Applied artifact SELECT policy scope is limited to these six tables:

| Table | Actual authority columns inspected | Payload or lineage fields | Default SELECT authority |
| --- | --- | --- | --- |
| `assess_processes` | `org_id`, `workspace_id`, `owner_id`, `status`, `archived_at`, `deleted_at` | none | non-null org/workspace, `deleted_at IS NULL`, active workspace membership |
| `assessments` | `org_id`, `workspace_id`, `process_id`, `created_by`, `updated_by`, `reviewer_id`, `approver_id`, `approved_by`, `rejected_by`, `status`, `archived_at`, `deleted_at` | `metadata`, `responses`, `evidence_items`, `assumptions`, `completion_by_section`, `review`, `scores` | non-null org/workspace, `deleted_at IS NULL`, active workspace membership, same-workspace process |
| `assessment_review_events` | `org_id`, `workspace_id`, `assessment_id`, `process_id`, `actor_id`, `event_type`, `status`, `created_at` | `payload`, `correlation_id` | non-null org/workspace, active workspace membership, same-workspace assessment and process |

`assessment_review_events` has no `deleted_at` column. Do not plan or implement a row-level `deleted_at` predicate for that table.

JSON fields such as `metadata`, `artifacts`, `payload`, `evidence_items`, `assumptions`, and `source_lineage` are payload or lineage fields only. They are never authorization authority.

## Default Authority Decision

M5.3a-2 should be workspace-scoped by default.

Default rules:

- `org_id IS NULL` fails closed.
- `workspace_id IS NULL` fails closed for all six artifact tables.
- `deleted_at IS NOT NULL` fails closed where `deleted_at` exists.
- `assessment_review_events` visibility relies on org/workspace and owning assessment/process authority because it has no `deleted_at`.
- `archived_at`, workflow `status`, `lifecycle_stage`, and `health_status` do not independently block SELECT in this slice unless AP later approves lifecycle-specific read restrictions.
- Source references never grant visibility.
- JSON payload fields never grant visibility.
- `source_lineage` never grants visibility.
- Cross-org and cross-workspace existence leaks must be denied.

Use `public.is_active_workspace_member(workspace_id, org_id)` by default. Use `public.is_active_org_member(org_id)` only if AP explicitly approves an org-only artifact visibility exception.

## Parent And Source Same-Workspace Hardening

Existing foreign keys often enforce same organization, not same workspace. Future policies must explicitly verify same workspace for parent and source references.

Required future same-workspace checks:

- `assessments.process_id` resolves to a same-organization, same-workspace, non-deleted `assess_processes` row.
- `assessment_review_events.assessment_id` resolves to a same-organization, same-workspace, non-deleted `assessments` row.
- `assessment_review_events.process_id` resolves to a same-organization, same-workspace, non-deleted `assess_processes` row.
- `projects.source_process_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assess_processes` row.
- `projects.source_assessment_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assessments` row.
- `document_generations.project_id` resolves to a same-organization, same-workspace, non-deleted `projects` row.
- `document_generations.source_process_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assess_processes` row.
- `document_generations.source_assessment_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assessments` row.
- `delivery_work_items.project_id` resolves to a same-organization, same-workspace, non-deleted `projects` row.
- `delivery_work_items.document_generation_id`, when present, resolves to a same-organization, same-workspace, non-deleted `document_generations` row.
- `delivery_work_items.source_process_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assess_processes` row.
- `delivery_work_items.source_assessment_id`, when present, resolves to a same-organization, same-workspace, non-deleted `assessments` row.

If a populated parent/source reference does not resolve under these same-workspace rules, the artifact row should fail closed.

## Future Policy Plan

Future implementation should create one migration using the actual next chronological timestamp. Based on the current migration chain, the planned filename is:

- `supabase/migrations/20260607153500_m5_3a_2_artifact_select_policies.sql`

Future migration shape:

- Enable RLS idempotently on the six artifact tables.
- Grant `SELECT` only to `authenticated` for these six tables.
- Drop and recreate one explicit `FOR SELECT TO authenticated` policy per table.
- Add policy comments documenting workspace fail-closed behavior, same-workspace parent/source checks, source-reference non-authority, JSON payload non-authority, no mutation scope, and no hosted-readiness claim.
- Do not create helper functions.
- Do not change M5.3a-1 helpers or policies.
- Do not create permission helpers.
- Do not create artifact mutation policies.

Per-table predicate direction:

- `assess_processes`: require non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, and active workspace membership.
- `assessments`: require non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, active workspace membership, and same-workspace non-deleted `process_id`.
- `assessment_review_events`: require non-null `org_id`, non-null `workspace_id`, active workspace membership, same-workspace non-deleted `assessment_id`, and same-workspace non-deleted `process_id`.
- `projects`: require non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, and active workspace membership. Populated source references must resolve same workspace.
- `document_generations`: require non-null `org_id`, non-null `workspace_id`, non-null `project_id`, `deleted_at IS NULL`, active workspace membership, and same-workspace non-deleted project. Populated source references must resolve same workspace.
- `delivery_work_items`: require non-null `org_id`, non-null `workspace_id`, non-null `project_id`, `deleted_at IS NULL`, active workspace membership, and same-workspace non-deleted project. Populated document/source references must resolve same workspace and non-deleted where applicable.
