# M4 Demo Polish Plan

## Goal

Prepare a focused demo showing the governed path from assessment to evidence-backed delivery.

## Scope

Demo script, sample story, seeded evidence, clear owner/risk/approval moments, and UI polish for existing flows.

## Explicit Out Of Scope

New product modules, runtime agents, compliance claims, and broad UI redesign.

## Files Allowed

Demo docs, seed data when approved, existing demo-facing UI files when approved, tests.

## Acceptance Criteria

Demo shows Process Intake -> Deterministic Assessment -> Operating Model -> Studio Docs -> Govern Card -> Approval/Evidence Rules -> Delivery Pack -> Monitor.

## Verification Commands

`npm run typecheck`, `npm run test`, `npm run build`, browser smoke check when UI changes are approved.

## Risks

Demo copy implying unsupported capability.

## Stop Conditions

Unsupported claims, new unapproved features, or UI changes outside demo scope.
