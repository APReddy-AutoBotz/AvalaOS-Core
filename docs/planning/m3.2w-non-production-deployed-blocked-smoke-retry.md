# M3.2w Non-Production Deployed Blocked-Smoke Retry

## Milestone Purpose

M3.2w retries the non-production deployed blocked-smoke path after the prior readiness milestones restored Supabase CLI availability, CLI authentication readiness, and database connectivity evidence.

The milestone is intentionally narrow: prove deployed fail-closed behavior only when the approved non-production target, pilot runtime mode, database readiness, deployed function readiness, and safe invocation prerequisites are all confirmed without leaking secrets.

## Scope

In scope:

- Confirm repository and baseline safety.
- Confirm required environment visibility using booleans only.
- Confirm `AVALA_AI_RUNTIME_MODE` is `pilot`.
- Confirm the approved non-production Supabase target without printing project ref, URL, host, token, key, auth header, database URL, username, password, or IP address.
- Run the existing M3.2p mocked DB smoke before deployed retry.
- List and deploy only the existing required Edge Functions if all preflight gates pass.
- Invoke only blocked deployed scenarios, plus safe mocked behavior only if it cannot call a live provider.
- Record sanitized evidence only.

Out of scope:

- Live Gemini or Groq calls.
- Real provider key use.
- Production, shared, customer, or unclear targets.
- Schema, RLS, migration, Admin UI, product UI, Health, scoring, browser fallback, runtime resolver, provider integration, Edge Function entrypoint, `_shared/ai.ts`, dependency, usage, or cost behavior changes.
- Unsupported compliance claims.

## Required Target Functions

M3.2w carries forward the M3.2t deployed blocked-smoke target functions:

- `ai-generate-document`
- `ai-refine-section`
- `ai-provider-test-connection`

These functions call provider execution only after resolver, audit, and secret gates allow the request. M3.2w may therefore exercise blocked paths. It must not run an allowed deployed scenario unless the existing deployed code provides a mock-only route that cannot call a live provider.
