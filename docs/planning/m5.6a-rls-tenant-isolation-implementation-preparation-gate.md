## Preparation Model

The M5.6a model distinguishes planned implementation surfaces from verified evidence. Every current proof status remains `unproven` or `evidence_required`; any future execution remains blocked until a separate AP-approved DB/RLS/artifact evidence gate defines exact scope, run count, output boundaries, prohibited outputs, stop conditions, and proof boundaries.

The model is conceptual only. It does not inspect the live schema, infer current table shape, apply policies, execute assertions, read rows, validate claims, or record tenant-isolation proof.
