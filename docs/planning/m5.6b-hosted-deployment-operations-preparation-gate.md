## Preparation Model

The M5.6b model distinguishes planned hosted/deployment/operations preparation from verified evidence. Every current proof status remains `unproven` or `evidence_required`. Any future execution remains blocked until a separate AP-approved evidence execution gate defines exact scope, run count, output boundaries, prohibited outputs, stop conditions, and proof boundaries.

The model is conceptual only. It does not probe environments, validate hosted targets, validate deployment targets, run startup checks, run readiness checks, execute rollback or incident behavior, execute backup/restore behavior, inspect schema, run DB/RLS/artifact checks, run provider/classifier behavior, or execute real assertions.
