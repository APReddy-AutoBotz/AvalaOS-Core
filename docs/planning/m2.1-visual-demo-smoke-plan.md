# M2.1 Visual Demo Smoke & Narrative Polish Plan

## Goal
Visually verify and narratively review the post-M2 AvalaOS Core golden path:

Avala Assess -> Avala Govern Lite -> Avala Studio -> Avala Delivery Pack -> Avala Monitor.

M2.1 is an evidence-only milestone. It creates planning, evidence, and findings documentation, and optionally screenshot evidence if local browser tooling is available. It does not change product behavior.

## Scope
- Visual smoke checklist.
- Demo story review.
- UI copy review.
- Navigation review.
- Export flow review.
- Month-End Close Evidence Pack review.
- Old branding review.
- Compliance-claim wording review.
- Jira replacement drift review.
- Narrative quality review.

## Out Of Scope
- New product features.
- UI fixes.
- Copy fixes.
- Backend hardening.
- Server-side AI changes.
- BYOK changes.
- Schema migrations.
- Runtime agents.
- MCP/A2A behavior.
- Health repo changes.
- Scoring changes.
- Package or dependency changes.
- Compliance claims.

## Evidence-Only Boundary
- Any visual, narrative, copy, navigation, or export issue found during M2.1 must be recorded in `docs/demo/m2.1-demo-polish-findings.md`.
- No issue may be fixed in M2.1.
- Product changes require a later AP-approved milestone.

## Golden Path Review Checklist
| Stage | Screen or Surface | Expected Narrative |
| --- | --- | --- |
| Avala Assess | Process Catalog, guided assessment, decision/recommendation surfaces | Evaluate first using deterministic assessment outputs and evidence. |
| Avala Govern Lite | Govern Lite card/panel | Govern before execution with owner, autonomy, risk, approval, evidence, allowed actions, and blocked actions visible. |
| Avala Studio | Docs Forge, generated document workspace, document vault | Generate editable delivery-ready documents from approved source context without implying raw confidential export. |
| Avala Delivery Pack | Project-scoped Delivery Pack under Avala Delivery Lite | Hand off with evidence, source lineage, work items, approval/evidence checklist, blockers, and local Markdown/JSON exports. |
| Avala Monitor | Portfolio and dashboard surfaces | Show portfolio value, risk, blockers, health, and decision status without certification language. |

## Visual Checks
- No overlapping text.
- No clipped buttons.
- No broken spacing.
- No illegible labels.
- No malformed separators or mojibake.
- Export buttons are visible and understandable.
- Delivery Pack tables/cards are readable on desktop and narrow widths.
- Avala Govern Lite clearly shows autonomy, risk, approval, and evidence.
- Monitor language is portfolio/value/risk oriented, not certification oriented.

## Navigation Checks
- Module Journey communicates lifecycle clearly.
- Delivery Pack remains under Avala Delivery Lite and project scope.
- No new top-level module appears.
- No Jira replacement drift appears.
- No old public branding appears in user-visible surfaces.

## Export Checks
- Markdown export can be triggered locally if browser tooling works.
- JSON export can be triggered locally if browser tooling works.
- Exported content uses references/summaries only.
- Export review checks for no raw uploaded content, document bodies, provider payloads, raw prompts, secrets, tenant-confidential bodies, or formal compliance/certification claims.

## Narrative Checks
- Demo proves: evaluate first, govern next, hand off with evidence.
- UI does not imply AI decides approvals, scores, risk gates, or compliance status.
- UI does not imply agents execute production actions.
- AvalaOS remains a governed AI and automation delivery platform, not generic PM, process mining, agent builder, or a Jira clone.

## Verification Commands
- `npm run typecheck`
- `npm run test`
- `npm run build`
- `npm audit --audit-level=moderate`
- `git diff --check`
- Approved wording scan for legacy names, prohibited compliance claims, and prohibited AI-decision phrasing.

## Stop Conditions
- Any product code needs change.
- Any scoring file needs change.
- Any Health file needs change.
- Any schema/package/dependency change is needed.
- Any copy implies AI decides approvals, scores, risk gates, or compliance status.
- Browser smoke reveals a P0 blocker requiring product changes.
- Old branding appears in user-visible surfaces.
