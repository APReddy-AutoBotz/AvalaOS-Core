# M4.1b Assess-to-Studio Full Handoff Payload Fix

## Purpose

M4.1b fixes critical finding `M4A-C02`: Avala Studio opened from Process Detail without the full read-only Assess source context. After M4.1a, Process Detail can render the current Decision Pack contract safely, but the `Generate BRD / PDD` handoff still passed only a thin set of identifiers and decision text.

This milestone makes Avala Studio receive a typed Assess source payload containing deterministic assessment context, Decision Pack context, Handoff Pack context, Govern Lite summary, evidence references, and assumptions.

## Scope

- Add a typed `AssessToStudioHandoffPayload`.
- Build the payload from the current process, scored assessment, Decision Pack, Handoff Pack, and Govern Lite summary.
- Pass the payload from Process Detail to `App.tsx` and into Docs Forge.
- Display the payload in Docs Forge as read-only source context.
- Attach the payload to generated artifacts so downstream milestones can consume it later.
- Record richer flat metadata in the existing handoff ledger entry without schema changes.
- Add focused regression coverage and wire it into `npm run test`.

## Non-Scope

- No deterministic scoring formula, weight, gate, risk tier, recommendation, or score-version change.
- No Docs-to-Delivery lineage or task import lineage implementation. That remains `M4.1c`.
- No Delivery Pack export guard. That remains `M4.1g`.
- No route/RBAC guard implementation.
- No schema, RLS, migration, or Supabase Edge Function change.
- No provider runtime or provider governance change.
- No Health change.
- No deployment, deployed function invocation, provider call, dependency addition, or secret access.

## Root Cause

The previous handoff shape was too thin for an evidence-backed Assess-to-Studio story. It included process and assessment identifiers plus a decision string, but not the current `DecisionPack`, `HandoffPack`, evidence references, assumptions, or Govern Lite summary that explain why Studio documents are grounded in the assessment.

## Payload Contract

`AssessToStudioHandoffPayload` is read-only source context with these field groups:

- Source identity: `sourceModule`, `targetModule`, `sourceType`, `sourceLabel`, `createdAt`.
- Process and assessment identity: `processId`, `processName`, `assessmentId`, `assessmentStatus`.
- Deterministic decision summary: `gateDecision`, `riskTier`, `confidenceBand`, `priorityTier`, `recommendationCategory`, `scoreVersion`, `calculatedAt`.
- Recommendation context: `operatingModelRecommendation`.
- Full source packs: `decisionPack`, `handoffPack`.
- Governance summary: `governLiteSummary`.
- Source references: `evidenceRefs`.
- Assumption register: `assumptionSummary`.
- Handoff readiness: `handoffEligibility`, `docsHandoffReadiness`, `deliveryHandoffReadiness`.

## User-Facing Behavior

- Process Detail enables `Generate BRD / PDD` only when a scored assessment is available.
- If scoring is missing, the action is disabled and labels that scoring is required before Studio handoff.
- Docs Forge shows an Assess source-context banner/card with decision status, risk, evidence count, assumption count, and document set.
- Studio includes the read-only Assess source context in the generation source material while preserving uploaded supporting files when provided.
- Generated artifacts keep `sourceContext` for future lineage work, but this milestone does not import or attach Delivery lineage.

## Handoff Ledger Behavior

The existing handoff ledger remains the persistence surface. M4.1b records richer flat metadata only:

- Process and assessment identifiers.
- Assessment status.
- Gate decision, risk tier, confidence band, priority tier, recommendation category.
- Score version and source label.
- Evidence reference count, assumption count, and required document types.

The ledger does not store nested payload objects and does not create new schema requirements.

## Tests

The focused regression verifies:

- Scored Assess-to-Studio payloads include Decision Pack and Handoff Pack context.
- Govern Lite summary, evidence references, assumptions, and document types are preserved.
- Missing optional Decision Pack / Handoff Pack fields do not crash payload construction.
- Unscored assessments return no handoff payload.
- Generated artifacts can carry the source context without mutating the original artifact object.

## Acceptance Criteria

- Process Detail no longer passes a thin handoff object to Studio.
- Docs Forge can receive and render Assess source context safely.
- The payload includes Decision Pack and Handoff Pack context when scores are available.
- Missing optional fields are safe.
- Existing M4.1a Decision Pack rendering regression still passes.
- Scoring regression still passes.
- M4.1c Docs-to-Delivery lineage is not implemented.
