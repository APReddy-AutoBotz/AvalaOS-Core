# M0.2 Build Control Pack Plan

## Goal

Create the operating rules, skills, governance docs, quality gates, ADRs, and evidence review needed for controlled future Codex work.

## Scope

Docs and `.agent` control files only.

## Explicit Out Of Scope

Runtime features, scoring changes, AI execution changes, Health changes, package changes, direct `main` push, and tag creation.

## Files Allowed

`AGENTS.md`, `.agent/**`, `docs/**`, and optional README links.

## Acceptance Criteria

- Required docs and skills exist.
- Legacy `.agent` files are inventoried and non-conflicting.
- Evidence review is completed after verification.

## Verification Commands

`npm run typecheck`, `npm run test`, `npm run build`, `npm audit --audit-level=moderate`, wording scan, changed-file review.

## Risks

Documentation bloat, conflicting rules, accidental product claims.

## Stop Conditions

Runtime file changes, scoring drift, Health changes, unsupported compliance claim, or old repo push.
