## Implementation Summary

M5.3a-2q adds the sanitized classification category `local-startup-timeout-with-oversized-output` to the existing local startup failure classifier.

The classifier now refines the safe fallback category `unknown-local-startup-failure` to `local-startup-timeout-with-oversized-output` only after building and validating the approved redacted signal fields. The classifier command interface remains unchanged and accepts only `--classify` and `--help`.

## New Category Summary

The new category is a redacted signal classification only. It is not:

- a root-cause claim
- a Docker claim
- a Supabase CLI claim
- a port claim
- a config claim
- a migration claim
- a local DB availability claim
- a hosted-pilot readiness claim
- a production readiness claim

The category is mapped to the existing `unknown` pattern family for sanitized family-count output.

## Exact Redacted Rule Summary

The classifier may emit `local-startup-timeout-with-oversized-output` only when the already-built redacted signal has all of these values:

- `outputSafetyResult=passed`
- `scratchCleanupResult=removed`
- `noSecretsConfirmation=passed`
- `noLocalPathConfirmation=passed`
- `noRawLogConfirmation=passed`
- `exitResultClass=timeout`
- `timeoutFlag=true`
- `lineCountBucket=oversized`
- `safetyBlockReasonCategory=output-too-large`

The classification refinement does not inspect raw stdout, raw stderr, raw logs, raw error lines, stack traces, local paths, host values, port values, IP values, DB URLs, identifiers, secrets, `.env` values, local config values, row payloads, auth headers, or claim values.

## Fallback Behavior Summary

The classifier keeps `unknown-local-startup-failure` if:

- any safety confirmation fails
- scratch cleanup is not `removed`
- timeout fields are incomplete
- `lineCountBucket` is not `oversized`
- `safetyBlockReasonCategory` is not `output-too-large`
- classification would require raw output
- classification would require local inspection
- classification would require local paths, ports, IDs, host values, DB URLs, secrets, logs, or target values

Existing fail-closed behavior remains in place for output-safety or scratch-cleanup failures.

## Confidence Behavior Summary

The implementation does not raise confidence above `low` for this new category. For the current exact redacted signal with `commandPhaseClass=unknown` and unknown pattern family count `1`, the category is `local-startup-timeout-with-oversized-output` and confidence remains `low`.
