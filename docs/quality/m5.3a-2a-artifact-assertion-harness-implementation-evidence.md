## Target Proof Behavior

The runner requires explicit local-target proof before any database action.

Accepted target forms:

- `localhost`
- `127.0.0.1`
- `loopback`, mapped to `127.0.0.1`

Rejected before database action:

- Hosted URLs.
- Remote hosts.
- Implicit environment database targets.
- Automatic database target execution from environment values.
- Secret-file fallback.
- Unknown arguments.
- Targets that cannot be proven local.

The runner does not print target values.

## SQL Static Guard Behavior

Before attempting database execution, the runner statically inspects the artifact assertion SQL.

Allowed assertion behavior:

- Transaction start.
- Transaction rollback.
- Transaction-scoped session settings.
- Temp tables.
- Transaction-scoped local fixture inserts.
- SELECT assertions.
- Local role/session reset behavior.
- Narrow temp-table grants to `authenticated` for `m5_3a_2_fixture_ids` and `m5_3a_2_scenario_results`.

Blocked assertion behavior:

- Transaction commit.
- Policy creation, alteration, or deletion.
- Durable table alteration, creation, or deletion.
- Function creation.
- Extension creation.
- Durable grants or revokes.
- Provider call patterns.
- Secret-looking target literals.

The narrow temp-table grant allowance is required by the existing assertion SQL so the local authenticated role can read fixture IDs and insert scenario result rows inside the rollback-scoped assertion transaction.

## Output Contract

Runner output is counts-only and sanitized.

Allowed output includes:

- Artifact scenario contract present yes/no.
- Assertion SQL present yes/no.
- Local target gate status.
- Local DB assertion execution status.
- Scenario counts.
- Pass, fail, and blocked counts.
- Artifact SELECT isolation verified yes/no.
- Harness status.
- Sanitized classifications.

Runner output must not include:

- Target values.
- Database URLs.
- Environment values.
- Privileged key values.
- Provider keys.
- JWT/private token values.
- Hosted URLs.
- Bucket names.
- Personal identifiers.
- Row payloads.
- Claim values.
- Auth headers.

The runner self-checks output before printing and fails with `output-sanitization-failed` if a forbidden output pattern is detected.
