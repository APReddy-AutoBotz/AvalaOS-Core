## Buyer-Facing Naming Changes

Buyer-facing lifecycle and module copy now presents:

`Avala Assess -> Avala Govern -> Avala Studio -> Avala Delivery -> Avala Monitor`

Changed buyer-facing labels:

- `Avala Govern Lite` -> `Avala Govern`
- `Govern Lite` -> `Avala Govern` or `Avala Govern control context`
- `Avala Delivery Lite` -> `Avala Delivery`
- `Delivery Lite` -> `Avala Delivery`

The wording still describes the implemented capability honestly:

- Avala Govern remains the Decision Pack, control card, human review, evidence, assumptions, and governed handoff context.
- Avala Delivery remains traceable work items, source lineage, evidence references, and Delivery Pack readiness.

## AvalaOS Brand Meaning Copy Changes

The buyer-facing login experience now includes:

`AvalaOS = Assess. Validate. Align. Launch. Audit.`

The primary demo story also records the same brand meaning line for demo talk-track consistency.

The preferred brand pattern is represented without adding new navigation, route behavior, module behavior, or product capability.
