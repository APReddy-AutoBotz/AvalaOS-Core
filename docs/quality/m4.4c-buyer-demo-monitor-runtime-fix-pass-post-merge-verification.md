## Root Cause

Root cause was traced to burndown chart division by zero when active-sprint story points are absent:

- Sarah's buyer `My Work` context had an active sprint but no active-sprint task estimates in the filtered task list.
- `totalStoryPoints` became `0`.
- The chart used `totalStoryPoints` as `maxY`.
- The Y-coordinate calculation divided by `maxY`, producing `0 / 0`.
- `NaN` then propagated into SVG text, line, path, and circle coordinates.
- The same chart model also needed an X-axis guard for one-day sprint windows.

## Fix Implemented

Confirmed merged fix:

- Extracted burndown chart math into `buildBurndownChartModel`.
- Added finite-coordinate guards for dimensions, sprint duration, story-point totals, chart points, axis ticks, and SVG paths.
- Returned `null` when a finite burndown model cannot be produced.
- Rendered a safe `No sprint estimates` empty state for active sprints without finite story-point estimates.
- Kept M4.4a Monitor wording intact.
