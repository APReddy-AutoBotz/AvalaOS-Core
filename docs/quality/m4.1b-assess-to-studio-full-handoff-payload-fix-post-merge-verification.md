## Root Cause

M4.0a critical finding `M4A-C02` found that the Assess-to-Studio handoff passed only thin process and assessment identifiers plus decision text. Docs Forge could open, but it did not receive the deterministic assessment context needed to ground generated documents in the current Decision Pack, Handoff Pack, Govern Lite summary, evidence references, assumptions, and readiness metadata.

## Handoff Payload Before And After

Before M4.1b:

- Process Detail passed a thin inline handoff object with `processId`, `processName`, optional `assessmentId`, and optional decision text.
- The handoff ledger received a basic event.
- Docs Forge did not render Assess source context.
- Generated artifacts did not retain Assess source context for future lineage work.

After M4.1b:

- Process Detail builds a typed `AssessToStudioHandoffPayload` from the current scored assessment.
- `App.tsx` stores the payload transiently, records richer flat handoff metadata, and passes the payload into Docs Forge.
- Docs Forge renders a read-only source-context card and includes the source context in generation source material.
- Generated artifacts retain a copied `sourceContext` for future M4.1c lineage/evidence work.

## Minimal Fix Summary

- Added shared Assess-to-Studio source-context types in `types.ts`.
- Added the pure helper `services/assessToStudioHandoff.ts`.
- Updated Process Detail so handoff requires a scored assessment and uses the typed payload.
- Updated Docs Forge to render and include read-only Assess source context.
- Attached source context to generated artifacts without schema changes.
- Recorded richer flat handoff ledger metadata without adding nested payload storage.
- Added focused regression coverage and wired it into `npm run test`.

## Source Context Fields Included

- Source identity: `sourceModule`, `targetModule`, `sourceType`, `sourceLabel`, `createdAt`.
- Process and assessment identity: `processId`, `processName`, `assessmentId`, `assessmentStatus`.
- Deterministic decision summary: `gateDecision`, `riskTier`, `confidenceBand`, `priorityTier`, `recommendationCategory`.
- Score metadata: `scoreVersion`, `calculatedAt`.
- Recommendation context: `operatingModelRecommendation`.
- Source packs: `decisionPack`, `handoffPack`.
- Governance context: `governLiteSummary`.
- Evidence and assumptions: `evidenceRefs`, `assumptionSummary`.
- Readiness: handoff eligibility, Docs handoff readiness, and Delivery handoff readiness.

## Stale Source-Context Hardening

- Valid Assess-to-Studio handoff remains the only path that sets `assessToStudioSourceContext`.
- Non-Assess Docs Forge entry paths clear transient Assess source context before opening Docs Forge.
- Docs Forge cancel clears the transient source context.
- Successful generation clears the transient source context after generated artifacts have received their own copied `sourceContext`.
- Manual or unrelated Docs Forge sessions cannot reuse a stale Assess source context from a previous process.

## Company Default Hardening

- Docs Forge no longer defaults the company field to `Avala Assess` when source context exists.
- Existing company/default behavior is preserved unless an actual organization/company value is safely available in a future approved milestone.
- The read-only card label `Avala Assess source context` remains because it identifies the source module, not the customer/company.
- No organization profile redesign was introduced.
