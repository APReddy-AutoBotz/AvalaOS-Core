## Implementation Summary

M5.3a-3k adds a narrow synthetic-only local stack readiness execution adapter layer.

The adapter is designed to connect a future AP-approved readiness command family to the existing M5.3a-3i wrapper contract, but this milestone does not run any real readiness or startup command. Execution is dependency-injected through a synthetic executor only.

No package script was added. The adapter has no direct child-process execution path and fails closed if no executor is supplied.

## Adapter Behavior Summary

The adapter imports and uses `scripts/m5.3a-3i-local-stack-readiness-procedure.mjs`.

The adapter accepts only the explicit synthetic command-family descriptor for this milestone:

- command family: `local-stack-readiness-procedure-synthetic`
- command label: `controlled-local-stack-readiness-procedure-run`
- synthetic-only: yes
- real execution approved: no

The adapter forwards only these sanitized fields from the synthetic executor into the M5.3a-3i wrapper:

- `runCount`
- `readinessAttemptStatus`
- `localStartupStatusBucket`
- `localStackReadinessBucket`
- `serviceReadinessBucket`
- `outputSafetyResult`
- `cleanupResult`
- `failClosedReason`

The adapter never records a literal command string and never emits shell command text.

The adapter fails closed when:

- the command-family descriptor is missing or not the approved synthetic descriptor;
- the executor is missing;
- the executor returns an unsupported field;
- the executor returns or requests raw stdout, stderr, log, stack, scratch, or raw-output-inspection data;
- the executor returns local path-like, DB URL-like, target-like, secret/token-like, hosted identifier-like, container/image ID-like, or machine-specific values;
- the executor requests more than one run;
- the executor implies root cause, schema readiness, RLS readiness, artifact readiness, tenant isolation, hosted readiness, or production readiness;
- cleanup fails;
- output safety fails;
- the M5.3a-3i wrapper returns fail-closed.
