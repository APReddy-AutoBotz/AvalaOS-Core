## Release Candidate Scope

M4.3 creates a buyer-demo release candidate evidence pack and two external review packs.

It is docs/evidence only. It does not implement new product features, UI changes, routes, views, data changes, scoring changes, schema changes, provider calls, deployment changes, or enterprise hardening.

Release candidate path:

`Avala Demo Enterprise -> AP Invoice Exception Handling / AP Invoice Exception Workflow -> Avala Assess -> Avala Govern -> Avala Studio -> Avala Delivery -> Avala Monitor`

Positioning:

- AvalaOS is buyer-demo ready for the governed AP workflow.
- AvalaOS is not yet enterprise-production deployed.
- Enterprise production readiness requires M5 Supabase/auth/RLS/audit/export/deployment hardening.

## Canonical Path Summary

The canonical AP story remains one coherent buyer-demo path:

1. Avala Assess evaluates AP Invoice Exception Handling.
2. Deterministic scoring produces fit, value, risk, and recommendation outputs.
3. Decision Pack explains rationale, controls, evidence, assumptions, and handoff readiness.
4. Avala Govern provides the control-card context for human review and governed handoff.
5. Avala Studio uses AP source context to present review artifacts.
6. Avala Delivery shows traceable AP workflow work items and Delivery Pack readiness.
7. Avala Monitor shows AP project/task/readiness signals.

## Buyer-Facing Lifecycle Summary

Buyer-facing lifecycle:

`Avala Assess -> Avala Govern -> Avala Studio -> Avala Delivery -> Avala Monitor`

This is a naming and evidence path. Avala Govern and Avala Delivery are not expanded beyond their current implemented maturity.
