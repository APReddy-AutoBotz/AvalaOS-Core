## Persisted State Risk Summary

Before M4.1d-4, restored `currentView` and `currentScope` values could be stale or malformed after browser storage, user, organization, module, permission, or scope changes.

Risk examples included:

- unknown or stale view strings,
- unknown scope types,
- malformed project or team scopes missing required fields,
- stale extra fields on simple scopes,
- clean-looking persisted view/scope pairs that the central guard should deny after access inputs changed.

## Sanitizer Summary

M4.1d-4 added `services/viewStatePersistence.ts` with pure persisted-state helpers:

- `isValidView`
- `normalizePersistedView`
- `isValidScope`
- `normalizePersistedScope`
- `areScopesEqual`
- `resolvePersistedViewScopeState`

Sanitizer behavior:

- Unknown/stale views normalize to Dashboard.
- Unknown scope types normalize to My Work scope.
- Malformed Project and Team scopes normalize to My Work scope.
- My Work and Organization scopes normalize to clean simple shapes.
- Valid Project and Team scopes preserve `type`, `id`, and `name`.
- Input objects are not mutated.
- The final normalized view/scope pair resolves through `resolveViewAccess`.

## App Integration Summary

`App.tsx` now keeps raw persisted `VIEW` and `SCOPE` values at the storage boundary and derives sanitized runtime `currentView` and `currentScope` values.

After auth and organization loading complete, App resolves the sanitized pair through the central View/RBAC guard and writes back normalized or fallback values only when needed.

The integration does not clear all browser storage and does not touch unrelated persisted keys.

## Fallback Behavior Summary

- Unknown or stale view values fall back to Dashboard, then resolve through the central guard.
- Unknown or malformed scope values fall back to My Work, then resolve through the central guard.
- Clean but guard-denied persisted combinations use the central guard fallback view/scope.
- Auth or organization loading waits instead of redirecting or mutating state from a loading condition.
- Existing Organization-scope Admin Workspace behavior remains preserved pending AP decisions.
