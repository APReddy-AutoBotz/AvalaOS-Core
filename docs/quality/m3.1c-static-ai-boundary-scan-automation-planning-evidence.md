## M3.1a / M3.1b Boundary Assumptions

- M3.1a introduced exactly four runtime modes: `local-demo`, `internal-dev`, `pilot`, and `production`.
- M3.1a makes `pilot` and `production` fail closed before browser fallback when Edge AI is unavailable.
- Client-side mode gating is a UI/orchestrator guard only; server-side Edge enforcement remains mandatory for pilot and production.
- M3.1b removed active runtime `userApiKey` generation/refinement paths and UI prop flow.
- M3.1b deleted `setup-api-key.js`.
- M3.1b removed browser-facing `vite.config.ts` provider-secret injection for `process.env.API_KEY` and `process.env.GEMINI_API_KEY`.
- `StorageKeys.API_KEY` remains as a deprecated cleanup-only path while `App.tsx` removes the legacy stored key.
- Browser provider classes and factories remain only for explicitly bounded `local-demo` / `internal-dev` fallback.
- M3.2 owns provider config and key-reference resolution.
