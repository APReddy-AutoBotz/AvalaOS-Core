## Timeout/Oversized-Output Fail-Closed Summary

Timeout plus oversized output can emit only the low-confidence redacted signal `local-startup-timeout-with-oversized-output`.

The timeout/oversized classifier logic accepts either:

- `outputSizeBucket = oversized`
- `lineCountBucket = oversized`

It still requires all safety gates:

- output safety passed
- scratch cleanup removed
- no-secrets confirmation passed
- no-local-path confirmation passed
- no-raw-log confirmation passed
- exit result is timeout
- `timeoutFlag = true`
- `safetyBlockReasonCategory = output-too-large`

Non-timeout over-budget output remains `unknown-local-startup-failure`. Unsafe output or cleanup failure still fails closed to `classification-output-safety-failed`. No root-cause inference was added.
