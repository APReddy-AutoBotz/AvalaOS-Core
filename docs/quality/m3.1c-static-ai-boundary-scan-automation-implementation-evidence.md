## Scan Design

- Added a zero-dependency Node script at `scripts/check-ai-boundary.mjs`.
- The script uses only built-in Node APIs.
- The script scans Git repository files through `git ls-files --cached --others --exclude-standard`.
- The script excludes generated/vendor folders such as `.git`, `.turbo`, `.vite`, `build`, `coverage`, `dist`, and `node_modules`.
- The script checks the approved AI boundary patterns as deterministic regular expressions.
- Every hit is classified by pattern, file path, and line number.
- A hit is forbidden unless it matches an exact allowlist entry or a controlled docs/evidence/control path category.
- Forbidden hits and stale exact allowlist entries fail the command with a nonzero exit code.
- Allowed hits, forbidden hits, and stale allowlist entries are printed separately.

The scanner intentionally scans tracked and untracked non-ignored repository files. Git-ignored local environment files are not part of the repository file set and are not scanned by default.

## Allowlist Structure

Allowlist file: `docs/quality/ai-boundary-static-scan-allowlist.json`

The allowlist contains:

- `schemaVersion`
- approved `allowedReasons`
- controlled `pathPrefixes` for planning/evidence/control paths
- exact runtime `entries` with:
  - `pattern`
  - `path`
  - `lines`
  - `reason`
  - `futureAction`

Allowed reasons are limited to:

- `deprecated cleanup-only path`
- `non-secret app localStorage state`
- `explicit local-demo/internal-dev fallback`
- `disabled surface`
- `server-side Edge/Supabase environment reference`
- `test/control documentation`
- `deferred M3.1c cleanup`
