## Local-Only Harness Recommendation

The verified plan recommends a future local-only harness milestone, M5.3a-1b, before helper or policy implementation. The future harness must:

- fail closed if a local-only target cannot be proven;
- reject remote database targets by design;
- avoid silent fallback to ignored secret files;
- avoid printing database targets, secrets, row payloads, tokens, or hosted identifiers;
- use generated local UUID fixtures only;
- report labels, scenario names, pass/fail status, counts, and sanitized diagnostics only.

## Remote-Target Rejection Design

The plan requires future harness code to accept only explicitly local targets such as localhost, loopback, Docker local network, or another AP-approved local-only target. It must not silently accept `DATABASE_URL`, hosted Supabase URLs, ignored local secret files, or remote project settings.
