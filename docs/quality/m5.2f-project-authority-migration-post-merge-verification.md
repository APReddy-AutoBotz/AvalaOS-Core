## Project Authority Semantics

`projects` represents an AvalaOS governed Delivery project under organization and workspace ownership.

Avala Portfolio remains a view over `projects` for now.

`projects` is not generic project-management replacement authority, not live runtime execution authority, and not bot execution authority.

## App ID Semantics

`app_id` is nullable and remains a UI/demo bridge only.

`app_id` is not tenancy authority and is not a Supabase project identifier.

`app_id` is not globally unique. Uniqueness is scoped by `(org_id, app_id)` and applies only when `app_id IS NOT NULL AND deleted_at IS NULL`.

## Constraints And Indexes Summary

The migration adds a composite helper key on `(id, org_id)` for future same-organization references.

Required semantics are staged through `NOT VALID` checks for `org_id`, non-empty `name`, and `status`.

Indexes cover organization/status/deleted lookup, workspace/status/deleted lookup, owner lookup, delivery lead lookup, source process lookup, source assessment lookup, and audit correlation lookup.

## Lifecycle And Status Check Decision

Lifecycle and status checks align with inspected runtime and draft values:

- `lifecycle_stage`: `Planning`, `Analysis & Design`, `Development`, `Testing`, `Deployment`, `Maintenance`
- `health_status`: `On Track`, `At Risk`, `Off Track`
- `status`: `active`, `paused`, `archived`, `completed`, `cancelled`

The checks are staged as `NOT VALID` to avoid breaking existing/manual non-production rows before AP-approved validation and backfill.

## FK Delete Behavior Summary

Tenant artifact ownership and source references avoid broad hard-delete cascade:

- `org_id` to `organizations(id)` uses default delete behavior.
- `(workspace_id, org_id)` to `workspaces(id, org_id)` uses default delete behavior.
- `(source_process_id, org_id)` to `assess_processes(id, org_id)` uses default delete behavior.
- `(source_assessment_id, org_id)` to `assessments(id, org_id)` uses default delete behavior.

Actor references may be nulled:

- `owner_id`, `delivery_lead_id`, `created_by`, and `updated_by` reference `profiles(id)` with `ON DELETE SET NULL`.

The evidence document records this tenant artifact history decision.
