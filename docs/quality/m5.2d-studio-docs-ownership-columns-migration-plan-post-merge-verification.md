## Studio/Docs Inventory Summary

`document_generations` is documented and referenced by runtime adapters, but it is not applied migration-chain authority. It exists in draft schema and current adapter flows only.

`document_generations` references `projects`, and `projects` is not yet applied migration-chain authority. This means Studio/Docs durable ownership depends on the future project/workspace boundary.

Generated artifacts remain runtime JSON through the `GeneratedArtifacts` shape. Studio source context remains runtime `AssessToStudioHandoffPayload`. Document Vault remains a UI/runtime concept only. Document versions, templates, document exports, Docs-to-Delivery lineage, handoff ledger authority, and AI generation jobs remain deferred to later approved milestones.

## Docs-Only Decision Summary

M5.2d remained docs-only. No Studio/Docs migration was created because implementing ownership columns now would require premature project/workspace authority decisions and could create unstable RLS inputs.

The records recommend planning Delivery/project ownership first, then returning to Studio/Docs ownership after the project/workspace boundary is approved.
