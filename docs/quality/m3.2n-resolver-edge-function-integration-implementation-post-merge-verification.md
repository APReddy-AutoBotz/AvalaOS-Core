## Resolver Integration Confirmation

Confirmed by file review and regression tests:

- The three integrated Edge Functions call `runProviderGovernedOperation` before AI job creation or provider handoff.
- Blocked resolver decisions fail closed with the sanitized response shape.
- Blocked decisions do not create AI jobs, do not resolve secrets, and do not call provider helpers.
- Allowed decisions require sanitized audit persistence before secret lookup.
- Provider calls receive explicit server-side key material only after resolver, audit, secret lookup, and scanner classification gates pass.
- Tests use mocked provider handoff behavior and do not run live provider calls.

## Secret Safety Confirmation

Confirmed by file review and focused integration tests:

- Secret lookup is available only after an allowed resolver decision and successful allowed-path audit persistence.
- Secret lookup accepts only provider-specific server environment reference names:
  - Groq: `AVALA_PROVIDER_SECRET_GROQ_*`
  - Gemini: `AVALA_PROVIDER_SECRET_GEMINI_*`
- Reserved platform, database, JWT, and direct provider-key environment names are rejected before any environment read.
- Wrong provider-prefix references are rejected.
- Secret lookup exceptions fail closed with sanitized response fields and `key_reference_ineligible`.
- Governance and secret lookup failure paths do not return raw DB/PostgREST error text, table names, stack traces, `secret_ref`, key material, auth headers, or provider payloads.
