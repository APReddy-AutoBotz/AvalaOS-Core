## Implementation Scope

Implemented SELECT-only policies for the six approved artifact authority tables:

- `assess_processes`
- `assessments`
- `assessment_review_events`
- `projects`
- `document_generations`
- `delivery_work_items`

The migration:

- Enables RLS idempotently on the six artifact tables.
- Revokes public/anon table access where those roles exist.
- Grants `SELECT` to `authenticated` on the six artifact tables.
- Revokes mutation-oriented table privileges from `authenticated`.
- Drops and recreates one explicit `FOR SELECT TO authenticated` policy per artifact table.
- Adds policy comments documenting workspace fail-closed behavior, same-workspace parent/source checks, source-reference non-authority, JSON payload non-authority, no mutation scope, and no hosted-readiness claim.

## Per-Table SELECT Policy Summary

`assess_processes`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires `deleted_at IS NULL`.
- Requires `public.is_active_workspace_member(workspace_id, org_id)`.

`assessments`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires `deleted_at IS NULL`.
- Requires active workspace membership.
- Requires `process_id` to resolve to a same-organization, same-workspace, non-deleted `assess_processes` row.

`assessment_review_events`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires active workspace membership.
- Requires `assessment_id` to resolve to a same-organization, same-workspace, non-deleted `assessments` row.
- Requires `process_id` to resolve to a same-organization, same-workspace, non-deleted `assess_processes` row.
- Does not use a row-level `deleted_at` predicate because `assessment_review_events` has no `deleted_at` column.

`projects`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires `deleted_at IS NULL`.
- Requires active workspace membership.
- Requires populated `source_process_id` to resolve to a same-organization, same-workspace, non-deleted `assess_processes` row.
- Requires populated `source_assessment_id` to resolve to a same-organization, same-workspace, non-deleted `assessments` row.

`document_generations`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires non-null `project_id`.
- Requires `deleted_at IS NULL`.
- Requires active workspace membership.
- Requires `project_id` to resolve to a same-organization, same-workspace, non-deleted `projects` row.
- Requires populated Assess source references to resolve same organization/workspace and non-deleted.
- Does not use `artifacts` JSONB for authorization.

`delivery_work_items`:

- Requires non-null `org_id`.
- Requires non-null `workspace_id`.
- Requires non-null `project_id`.
- Requires `deleted_at IS NULL`.
- Requires active workspace membership.
- Requires `project_id` to resolve to a same-organization, same-workspace, non-deleted `projects` row.
- Requires populated `document_generation_id` to resolve to a same-organization, same-workspace, non-deleted `document_generations` row.
- Requires populated Assess source references to resolve same organization/workspace and non-deleted.
- Does not use `source_lineage` or `metadata` JSONB for authorization.

## Parent And Source Same-Workspace Hardening

The policies explicitly add same-workspace checks because existing foreign keys mostly prove same organization, not same workspace.

Populated parent/source references fail closed unless they resolve to the same `org_id` and `workspace_id` as the artifact row. Deleted parent/source rows also fail closed where `deleted_at` exists on the referenced table.

## Nullable Org And Workspace Fail-Closed Behavior

All six artifact policies require:

- `org_id IS NOT NULL`
- `workspace_id IS NOT NULL`

No org-only artifact path was introduced. The migration does not use `public.is_active_org_member(org_id)` for artifact visibility.

## Deleted-At Behavior

Rows with `deleted_at IS NOT NULL` fail closed where `deleted_at` exists:

- `assess_processes`
- `assessments`
- `projects`
- `document_generations`
- `delivery_work_items`

`assessment_review_events` has no `deleted_at` column. Its policy relies on non-null org/workspace, active workspace membership, and same-workspace non-deleted owning assessment/process authority.

## JSON, Payload, And Source Lineage Non-Authority

The migration does not use JSON payload fields as authorization inputs. This includes:

- `metadata`
- `artifacts`
- `payload`
- `source_lineage`
- embedded assessment evidence/assumption/review/score JSON fields

Source references are lineage only and never independently grant visibility.
