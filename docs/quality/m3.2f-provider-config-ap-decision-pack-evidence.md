## Guardrails

- Raw provider secrets must not be stored in ordinary application DB rows, browser env, browser state, logs, evidence, responses, UI-facing payloads, or product copy.
- Raw provider secrets must not be stored encrypted in ordinary application DB rows unless AP separately approves a dedicated secret-storage model.
- Key references should point to server-side secret material or secret manager identifiers only.
- Browser/client code must never resolve key references or receive raw keys.
- Provider audit events must prove gates were checked without becoming sensitive payload storage.
- Missing, ambiguous, disabled, denied, or unsafe provider config/policy/key-reference/prompt/evidence states must fail closed.
- Future implementation remains AP-gated.
