### Path A: Real Binding-To-Adapter Integration Implementation

Add a future implementation milestone that lets the existing adapter recognize the metadata-only real command binding descriptor.

### Path B: Literal Command-Family Design Gate

Create a future docs-only gate to decide whether AP will approve a literal local readiness command family.

Path B is required before any actual command can be executed. It must still prohibit raw output, local paths, secrets, DB URLs, container or image IDs, and machine-specific evidence.

M5.3a-3n does not approve Path B execution, does not approve any literal command string, and does not decide the content of a future literal command family.

### Path C: Stop Local Readiness Track

Stop here because the repo now has a wrapper, an adapter, and a metadata-only binding, but no real command execution approval.

Path C is recommended if AP does not want to spend more cycles on local readiness.

### Path D: Hosted Or Nonlocal Validation Track

Defer local readiness and create separate hosted or nonlocal validation boundaries.

Path D requires separate AP approval and separate proof language. It does not prove local readiness.

## Recommended Path

Recommended path: Path A only if AP wants to continue toward local readiness evidence.

M5.3a-3n does not approve literal command design and does not approve real execution.

If AP does not want further local readiness work, choose Path C.
