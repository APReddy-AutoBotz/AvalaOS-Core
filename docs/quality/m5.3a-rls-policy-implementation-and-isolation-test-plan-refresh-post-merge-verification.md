Default future policy posture:

- Deny by default.
- SELECT-only first slice.

- Mutations denied until later AP approval.
- Service-role behavior server-only and never browser-callable.

- Grant execute only to intended authenticated roles.
- Prevent browser service-role exposure.
- Prevent cross-tenant leakage.
