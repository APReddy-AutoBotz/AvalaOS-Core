# M5.3a-3a Controlled Evidence Rerun Approval Gate

## Milestone

M5.3a-3a Controlled Evidence Rerun Approval Gate.

## Execution Type

Docs-only approval gate.

This milestone records the exact AP approval requirements for a future controlled classifier evidence rerun after the M5.3a-2z safe capture remediation. It does not run the classifier and does not approve a future rerun by itself.

M5.3a-2z Fail-Closed Evidence Capture Remediation Implementation is closed and tagged.

## Future Rerun Scope

Future milestone name:

- `M5.3a-3b Controlled Evidence Rerun After Capture Remediation`

Future command label:

- `controlled-evidence-rerun-after-capture-remediation`

Future run count:

- Exactly one classifier classification evidence run.

Future execution permission:

- Not approved by this PR.
- Requires explicit AP approval after M5.3a-3a is merged, verified, and tagged.

## Future Allowed Execution

In M5.3a-3b only, after explicit AP approval:

- classifier classification mode may run exactly once;
- classifier execution may indirectly attempt local startup;
- output must be processed only through the M5.3a-2z capture helper;
- evidence must record only the approved command label, not a literal command string.

## Future Prohibited Execution

M5.3a-3b must still prohibit:

- direct startup command;
- Supabase stack command;
- Docker command or listing;
- DB, schema, RLS, artifact, migration, or bootstrap command;
- hosted validation;
- provider call;
- second classifier run;
- raw output inspection.

## Future Allowed Evidence

M5.3a-3b evidence may record only:

- run count;
- command label only;
- capture helper status;
- sanitized reason category if capture fails closed;
- sanitized captured fields if capture succeeds;
- output safety result;
- scratch cleanup result;
- proof boundaries;
- local DB availability outcome only as a sanitized field, without inference;
- schema availability outcome only as a sanitized field, without inference;
- artifact SELECT isolation outcome only as a sanitized field, without inference;
- tenant isolation outcome only as a sanitized field, without inference.

## Future Prohibited Evidence

M5.3a-3b evidence must not include:

- literal command string;
- raw stdout, stderr, or logs;
- scratch output;
- stack traces;
- local paths;
- `.env` values;
- host, port, IP, or DB URL values;
- target values;
- row payloads;
- auth headers;
- claim values;
- provider keys;
- service-role values;
- private tokens;
- container IDs or image IDs;
- hosted identifiers;
- machine-specific values.

## Future Stop And Fail-Closed Rules

M5.3a-3b must stop and record sanitized fail-closed evidence if:

- the M5.3a-2z capture helper returns fail-closed;
- raw output would need inspection;
- a second classifier run would be needed;
- local details or secrets could be exposed;
- the result would require root-cause inference;
- the result would overclaim DB, schema, RLS, artifact, hosted, or production readiness.

## Proof Boundaries

- M5.3a-3a is docs-only.
- No classifier category is newly proven.
- No evidence rerun is performed.
- Root cause is not inferred.
- Local DB availability remains unresolved.
- Schema availability is not proven.
- Artifact SELECT isolation is not verified.
- Tenant isolation is not newly verified.
- Hosted readiness is not proven.
- Production readiness is not proven.

## AP Decisions Required Before M5.3a-3b

AP must explicitly approve:

- that M5.3a-3b may perform exactly one classifier classification evidence run;
- the command label `controlled-evidence-rerun-after-capture-remediation`;
- continued use of the M5.3a-2z capture helper as the only evidence capture path;
- stopping on capture helper fail-closed;
- stopping if raw output inspection would be required;
- stopping if local details, secrets, or machine-specific values could be exposed;
- stopping if a second classifier run would be needed;
- the allowed evidence fields and prohibited evidence fields listed in this gate;
- the continued ban on direct startup, Supabase stack, Docker, DB, schema, RLS, artifact, migration, bootstrap, hosted validation, and provider commands outside the approved classifier classification path.

## M5.3a-3a Non-Scope Confirmation

M5.3a-3a does not:

- run a classifier classification evidence path;
- rerun evidence;
- attempt startup;
- run Supabase stack commands;
- run Docker commands;
- open a DB connection;
- run schema, RLS, artifact, migration, or bootstrap commands;
- run hosted validation;
- run provider calls;
- change code, tests, package files, CI, Supabase files, SQL, migrations, RLS policies, artifact harnesses, provider files, runtime files, UI files, or deployment files;
- start M5.3a-3b.

## Exposure Boundary

M5.3a-3a does not read, print, summarize, copy into evidence, expose, or commit secrets, target values, DB URLs, row payloads, auth headers, claim values, `.env` values, hosted identifiers, provider keys, service-role values, private tokens, container IDs, image IDs, raw logs, scratch output, local config values, prohibited classifier/startup command strings, or local machine paths.
