## Helper Function Design Proposal

Future required helpers:

- `public.is_active_org_member(p_org_id uuid)`
- `public.is_active_workspace_member(p_workspace_id uuid, p_org_id uuid)`

Design direction:

- `LANGUAGE sql`
- `STABLE`, not `IMMUTABLE`
- `SECURITY DEFINER` only if required to avoid RLS recursion
- hardened `search_path`
- schema-qualified references wherever possible
- boolean output only
- no tenant metadata output
- no role existence output
- no browser service-role exposure
- execute revoked from `PUBLIC` and `anon` where applicable
- execute granted only to `authenticated`

Predicates require `auth.uid()`, matching active non-deleted membership, active non-deleted parent organization, and active non-deleted parent workspace where workspace applies.

## SELECT Policy Plan

Future SELECT policy direction:

- `profiles`: own active, non-deleted profile only.
- `organization_members`: own active, non-deleted membership only, gated by active org helper.
- `workspace_memberships`: own active, non-deleted workspace membership only, gated by active workspace helper.
- `organizations`: readable only through active org membership.
- `workspaces`: readable only through active workspace membership.
- `roles`: org roles through active org membership; workspace roles through active workspace membership; system role visibility deferred to AP.

The plan explicitly excludes broad `FOR ALL`, INSERT, UPDATE, DELETE, hard DELETE, mutation, artifact, provider governance, and legacy draft table policies.

## Recursion Risk Analysis

Risks recorded:

- Helpers may query membership tables while those tables have RLS enabled.
- Membership policies may depend on helpers.
- Role policies may depend on membership visibility.
- Overbroad definer helpers can leak membership or role existence across tenants.

Controls recorded:

- Minimal helper functions only.
- No permission helpers in M5.3a-1.
- No role-helper dependency in membership visibility.
- No artifact table dependency.
- Local validation required before verified isolation can be claimed.

## Bootstrap And Access Paradox

The plan allows the minimum read path required for users to operate:

- Own profile read.
- Own active organization membership read.
- Own active workspace membership read.
- Organization metadata through active membership.
- Workspace metadata through active membership.
- Scoped role metadata through active membership.

The plan blocks cross-tenant membership and role existence leaks, disabled/deleted membership bootstrap, and system role visibility unless AP approves it.
