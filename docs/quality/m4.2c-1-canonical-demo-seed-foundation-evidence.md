## Canonical Seed Summary

M4.2c-1 establishes the first safe canonical AP story foundation:

- Organization: `Avala Demo Enterprise`
- Scenario: `AP Invoice Exception Handling`
- Workflow label: `AP Invoice Exception Workflow`
- Domain: `Finance Operations / Accounts Payable`
- Users and roles:
  - Platform Admin
  - AP Process Owner
  - Process Analyst
  - Delivery Lead
  - Control Reviewer
  - Automation Contributor
  - Buyer Viewer
- Enabled modules: Assess, Studio, Delivery, and Monitor.
- Process Catalog now exposes one canonical completed AP process.
- Guided Assessment seed includes completed responses, evidence, assumptions, owner review notes, approval history, and deterministic score output.
- Decision Pack and Handoff Pack source data are generated through the existing scoring engine and current score version.
- Govern Lite source data is derived through the existing Govern Lite service from the canonical assessment/process.
- The Process Analyst is the first mock user so default local demo entry aligns to the AP Assess walkthrough.

## Canonical Evidence And Assumptions

The canonical assessment includes demo evidence only:

- AP exception current-state map.
- AP exception handling SOP.

The canonical assessment includes assumptions for volume, time and motion, system access, governance, and data quality. These assumptions are marked as demo assumptions and are linked to existing assessment fields.
