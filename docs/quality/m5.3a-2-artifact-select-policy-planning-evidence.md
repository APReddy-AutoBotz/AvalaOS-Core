## Default Authority Decision

Future artifact SELECT policy planning uses these defaults:

- `org_id IS NULL` fails closed.
- `workspace_id IS NULL` fails closed for all six artifact tables.
- `deleted_at IS NOT NULL` fails closed where `deleted_at` exists.
- `assessment_review_events` has no `deleted_at`; owning assessment/process authority supplies deletion visibility checks.
- `archived_at`, `status`, `lifecycle_stage`, and `health_status` do not independently block SELECT in this slice.
- Source references never grant visibility.
- JSON payload fields never grant visibility.
- `source_lineage` never grants visibility.
- Active workspace membership is required by default.

## Per-Table Policy Plan

`assess_processes`:

- SELECT requires non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, and `public.is_active_workspace_member(workspace_id, org_id)`.

`assessments`:

- SELECT requires the same base checks.
- `process_id` must resolve to same `org_id`, same `workspace_id`, and non-deleted `assess_processes`.

`assessment_review_events`:

- SELECT requires non-null `org_id`, non-null `workspace_id`, and active workspace membership.
- `assessment_id` and `process_id` must resolve to same `org_id`, same `workspace_id`, and non-deleted owning rows.
- No row-level `deleted_at` predicate is planned because the table has no `deleted_at`.

`projects`:

- SELECT requires non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, and active workspace membership.
- Populated source references must resolve same workspace.

`document_generations`:

- SELECT requires non-null `org_id`, non-null `workspace_id`, non-null `project_id`, `deleted_at IS NULL`, and active workspace membership.
- `project_id` must resolve to same `org_id`, same `workspace_id`, and non-deleted `projects`.
- Populated source references must resolve same workspace.
- `artifacts` is payload only.

`delivery_work_items`:

- SELECT requires non-null `org_id`, non-null `workspace_id`, non-null `project_id`, `deleted_at IS NULL`, and active workspace membership.
- `project_id` must resolve to same `org_id`, same `workspace_id`, and non-deleted `projects`.
- Populated `document_generation_id` must resolve same workspace and non-deleted.
- Populated source references must resolve same workspace.
- `source_lineage` and `metadata` are payload only.
