## Output-Budget Controls Added

The classifier now defines named output-budget controls:

- `SAFE_OUTPUT_BYTE_BUDGET`
- `SAFE_OUTPUT_LINE_BUDGET`
- `STARTUP_CAPTURE_MAX_BUFFER_BYTES`
- `STARTUP_TIMEOUT_MS`

The startup subprocess path now uses an explicit `maxBuffer` value tied to the safe byte budget. Bounded capture metadata records sanitized byte and line-count buckets plus an internal output-budget pressure flag.

When output-budget pressure is detected, raw startup output is not carried forward for classification pattern matching. Scratch capture uses a redacted marker instead of raw oversized output.

The implementation also treats max-buffer exhaustion as sanitized output-budget pressure without printing or copying raw output.

## Timeout/Oversized-Output Fail-Closed Behavior Added

Timeout plus oversized-output or output-budget pressure now resolves through redacted signal metadata only.

If the safe redacted signal has:

- timeout exit result,
- `timeoutFlag=true`,
- oversized line-count bucket,
- output-too-large safety block category,
- passed output safety,
- removed scratch cleanup,
- passed no-secrets, no-local-path, and no-raw-log confirmations,

then the classifier can emit the existing low-confidence signal category:

`local-startup-timeout-with-oversized-output`

This remains a redacted signal category only. It is not a root-cause claim.

Over-budget non-timeout output remains `unknown-local-startup-failure` and does not become a timeout or root-cause claim.

If sanitized output cannot be produced safely, the classifier fails closed to:

`classification-output-safety-failed`
