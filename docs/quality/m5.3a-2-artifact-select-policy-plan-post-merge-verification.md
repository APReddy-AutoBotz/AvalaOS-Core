## M5.3a-1 Accepted Baseline

M5.3a-1 is the closed prerequisite. It implemented two boolean membership helpers:

- `public.is_active_org_member(uuid)`
- `public.is_active_workspace_member(uuid, uuid)`

The helpers are `SECURITY DEFINER`, `STABLE`, boolean-only, hardened with `search_path = pg_catalog`, schema-qualified, execute-revoked from `PUBLIC` and `anon`, and execute-granted only to `authenticated`.

M5.3a-1 implemented SELECT-only policies for `profiles`, `organizations`, `workspaces`, `roles`, `organization_members`, and `workspace_memberships`. Role visibility explicitly requires `is_system = false`. No artifact policies existed before M5.3a-2, and none were added by M5.3a-2.

## Implementation Scope Confirmation

Future implementation scope is SELECT-only policies for these six artifact tables:

- `assess_processes`
- `assessments`
- `assessment_review_events`
- `projects`
- `document_generations`
- `delivery_work_items`

The plan defers INSERT, UPDATE, DELETE, soft-delete, review, approval, export, delivery, and mutation policies. It also defers provider governance policies, storage/export policies, legacy draft table policies, runtime adapters, UI behavior, CI/deployment changes, and Health repository work.

## Default Authority Decision Confirmation

The accepted plan records these default decisions:

- `org_id IS NULL` fails closed.
- `workspace_id IS NULL` fails closed for all six artifact tables.
- `deleted_at IS NOT NULL` fails closed where `deleted_at` exists.
- `assessment_review_events` has no `deleted_at` column; no row-level `deleted_at` predicate should be planned for that table.
- `archived_at`, `status`, `lifecycle_stage`, `health_status`, `priority`, and `type` are not authorization gates in this slice unless AP later approves lifecycle-specific read restrictions.
- M5.3a-2 uses active workspace membership by default through `public.is_active_workspace_member(workspace_id, org_id)`.
- `public.is_active_org_member(org_id)` is reserved only for future AP-approved org-only exceptions; no org-only artifact path is approved by this plan.

## Per-Table Policy Plan Confirmation

The planning pack confirms:

- `assess_processes`: require non-null `org_id`, non-null `workspace_id`, `deleted_at IS NULL`, and active workspace membership.
- `assessments`: require process authority plus `process_id` resolving to a same-org/same-workspace non-deleted `assess_processes` row.
- `assessment_review_events`: require non-null org/workspace, active workspace membership, and both `assessment_id` and `process_id` resolving to same-org/same-workspace owning rows; no row-level deleted predicate exists for the event row.
- `projects`: require non-null org/workspace, `deleted_at IS NULL`, active workspace membership, and same-workspace validation for populated source refs.
- `document_generations`: require non-null org/workspace/project, `deleted_at IS NULL`, active workspace membership, same-org/same-workspace non-deleted project, and same-workspace validation for populated source refs.
- `delivery_work_items`: require non-null org/workspace/project, `deleted_at IS NULL`, active workspace membership, same-org/same-workspace non-deleted project, and same-workspace validation for populated document/source refs.

## Parent And Source Hardening Confirmation

Same-organization foreign keys do not prove same-workspace isolation. Future policies must explicitly verify same workspace for populated parent/source references. Populated parent/source reference failure must fail closed.

Source references are never authorization authority. `source_lineage` is never authorization authority.
