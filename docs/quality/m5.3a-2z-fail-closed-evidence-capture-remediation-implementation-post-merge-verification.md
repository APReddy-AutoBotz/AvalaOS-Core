## Synthetic-Only Parser And Capture Behavior

The parser accepts only approved sanitized top-level fields and redacted signal fields.

Valid capture requires:

- all required top-level fields;
- all required redacted signal fields;
- allowlisted enum values;
- passed output safety;
- non-failed scratch cleanup;
- passed no-secrets, no-local-path, and no-raw-log confirmations;
- no root-cause implication;
- no readiness implication.

The helper rejects literal command strings and does not accept raw output, scratch output, generated logs, local paths, local machine state, target values, DB URLs, row payloads, auth headers, claim values, provider keys, service-role values, private tokens, container IDs, or image IDs as evidence fields.
