Work items, Delivery Pack, Monitor, lineage/handoff, and export ownership remain deferred until project authority exists.

Future work-item authority should add only needed project/workspace ownership, assignee/reporter actors, source lineage refs, lifecycle/status, and correlation fields.

## FK And Backfill Strategy Summary

No `NOT NULL` ownership requirements, seed data, broad updates, personal IDs, hosted IDs, or destructive backfill should be introduced before AP-approved mapping.
