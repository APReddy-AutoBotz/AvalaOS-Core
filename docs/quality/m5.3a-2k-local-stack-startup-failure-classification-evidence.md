## Classifier Implementation Summary

M5.3a-2k adds a controlled classifier script for the previously failed local-only startup path.

The classifier:

- Accepts only `--classify` and `--help`.
- Runs and consumes the existing static M5.3a-2h diagnostic categories before any startup attempt.
- Does not run startup if sanitized preconditions are not met.
- Runs exactly one local-only `supabase start` attempt only when diagnostic readiness is `ready`.
- Captures raw stdout and stderr into generated scratch output only.
- Never prints raw startup output.
- Classifies raw output internally using allowlisted patterns.
- Emits only sanitized category fields.
- Deletes generated scratch output before exit whenever possible.
- Exits non-zero if output safety fails or scratch cleanup fails.
