## Migration Summary

M5.2c adds the first Assess persistence and ownership slice needed for future workspace-aware RLS. The migration is additive and promotes only three Assess tables into the migration chain:

- `assess_processes`
- `assessments`
- `assessment_review_events`

The migration uses `CREATE EXTENSION IF NOT EXISTS pgcrypto`, `CREATE TABLE IF NOT EXISTS`, `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`, guarded `DO $$` blocks, and `CREATE INDEX IF NOT EXISTS`.

## Assess-First And Govern-Deferred Clarification

- Assess base persistence and ownership inputs are implemented.
- Standalone Govern persistence is deferred.
- Standalone Decision Pack persistence is deferred.
- Govern remains runtime-derived through Decision Pack rendering, exports, and UI panels until a later AP-approved persistence milestone.

## Ownership Columns Summary

`assess_processes` supports:

- `org_id`
- nullable `workspace_id`
- `owner_id`
- `created_by`
- `updated_by`
- `archived_at`
- `deleted_at`

`assessments` supports:

- `org_id`
- nullable `workspace_id`
- `process_id`
- `created_by`
- `updated_by`
- `reviewer_id`
- `approver_id`
- `approved_by`
- `rejected_by`
- `audit_correlation_id`
- `archived_at`
- `deleted_at`

`assessment_review_events` supports:

- `org_id`
- nullable `workspace_id`
- `assessment_id`
- `process_id`
- `actor_id`
- `correlation_id`

No `source_org_id` or `source_workspace_id` columns were added in this slice.

## Constraints And Indexes Summary

- Helper unique constraints on `(id, org_id)` are added for `assess_processes` and `assessments`.
- Required-column and status checks are guarded and broad enough for existing/manual rows.
- Same-org composite FK direction is added with `NOT VALID` constraints where appropriate.
- Lookup indexes cover org, workspace, status, deleted markers, process, assessment, actor, reviewer, approver, and correlation fields.

## Same-Org FK Protection Summary

- `assessments(process_id, org_id)` references `assess_processes(id, org_id)` using a `NOT VALID` constraint.
- `assessment_review_events(assessment_id, org_id)` references `assessments(id, org_id)` using a `NOT VALID` constraint.
- `assessment_review_events(process_id, org_id)` references `assess_processes(id, org_id)` using a `NOT VALID` constraint.
- `(workspace_id, org_id)` references `workspaces(id, org_id)` for nullable workspace ownership.
- The `NOT VALID` constraints are not validated in this milestone.

## FK And Backfill Strategy

- Ownership columns are nullable-first.
- `workspace_id` is nullable and not backfilled in M5.2c.
- No `NOT NULL` ownership enforcement is added for existing/manual rows.
- No seed data is added.
- No backfill is performed.
- No committed hosted organization, workspace, project, bucket, or personal values are introduced.
- Canonical AP demo workspace mapping remains a later AP-approved task.
