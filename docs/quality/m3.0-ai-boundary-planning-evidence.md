## Risks Identified

- Browser provider code remains present for transitional demo/internal-dev behavior.
- Pilot/production behavior needs explicit mode gating so browser-side provider execution and browser-stored raw keys are forbidden.
- Provider config and key-reference schema exists, but Edge provider resolution is not yet fully wired to it.
- Prompt bodies currently live in code and are not consistently recorded as prompt key/version evidence.
- AI job and usage audit exists, but pilot fail-closed enforcement is not yet proven.
- Client-submitted usage logging requires later review before pilot.
- Current UI accepts PDF/DOCX uploads, but current safe extraction is not production-ready for binary formats.
- Admin AI Controls readiness is not yet visible as a read-only status surface.
- Any schema/RLS need in later slices must stop for AP approval.

## Mode Matrix

| Mode | AI Execution | Required Boundary |
| --- | --- | --- |
| `local-demo` | Prepared demo data and synthetic-only fallback allowed. | No real customer data. |
| `internal-dev` | Transitional fallback may exist only when clearly labeled. | Must not be used for real customer data. |
| `pilot` | Server-side Edge AI only. | Fail closed on missing provider config, key reference, auth, org membership, prompt version, or usage audit. |
| `production` | Server-side Edge AI only. | Same as pilot, with no browser raw keys or browser-side provider execution. |
