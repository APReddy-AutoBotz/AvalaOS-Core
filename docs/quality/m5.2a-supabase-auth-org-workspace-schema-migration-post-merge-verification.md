## Migration Summary

The M5.2a migration adds the first Supabase Auth / organization / workspace tenant-authority schema foundation. It is additive and non-destructive, includes `CREATE EXTENSION IF NOT EXISTS pgcrypto`, uses `CREATE TABLE IF NOT EXISTS` for the approved authority tables, uses idempotent column/index/constraint patterns where compatibility is needed, and does not change application runtime behavior.

## Tables Created

- `organizations`

- `roles`
- `organization_members`
- `workspace_memberships`

No other table class was added by M5.2a.

## Constraints And Indexes Summary

- `profiles`: `id` references `auth.users(id)` and active/non-deleted lower email uniqueness is indexed.
- `organizations`: tenant-root metadata with active/non-deleted lower slug uniqueness.
- `workspaces`: organization child authority with active/non-deleted lower slug uniqueness per organization and a unique `(id, org_id)` helper key.
- `roles`: system, organization, and workspace scope checks with JSONB permissions retained for compatibility.
- `organization_members`: active/non-deleted unique `(org_id, user_id)` organization membership authority.
- `workspace_memberships`: composite workspace/organization foreign key, active/non-deleted unique `(workspace_id, user_id)`, and a non-unique `(org_id, user_id)` lookup index.

## Workspace Membership Amendment Confirmation

- `workspace_memberships` has active/non-deleted uniqueness by `(workspace_id, user_id)`.
- `workspace_memberships` has a non-unique lookup index on `(org_id, user_id)`.
- No unique constraint or unique index exists on `workspace_memberships(org_id, user_id)`.
