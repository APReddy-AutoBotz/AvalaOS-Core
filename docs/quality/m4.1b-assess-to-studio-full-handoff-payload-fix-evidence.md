## Root Cause

M4.0a critical finding `M4A-C02` found that Assess-to-Studio handoff did not carry the full assessment source context into Docs Forge. The prior Process Detail callback passed only process id, process name, optional assessment id, and decision text. Studio could open, but the source context did not include the current Decision Pack, Handoff Pack, evidence references, assumptions, or governance summary needed for evidence-backed document drafting.

## Handoff Payload Before / After

Before M4.1b:

- Process Detail passed a thin inline object with `processId`, `processName`, optional `assessmentId`, and optional decision text.
- `App.tsx` recorded a basic handoff ledger entry.
- Docs Forge did not receive or render Assess source context.
- Generated artifacts did not retain source context for a future lineage milestone.

After M4.1b:

- Process Detail builds a typed `AssessToStudioHandoffPayload` from the scored assessment.
- `App.tsx` stores the payload in handoff state, records richer flat ledger metadata, and passes the payload to Docs Forge.
- Docs Forge renders a read-only Assess source-context card and includes the context in source material for generated drafts.
- Generated artifacts retain `sourceContext` for future M4.1c lineage work.
