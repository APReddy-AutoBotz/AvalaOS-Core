## Secret Hygiene Rule Summary

The new `scripts/check-secret-hygiene.mjs` script checks these high-confidence rules:

- `tracked-env-file`: tracked or unignored `.env*` paths are forbidden by path only.
- `server-only-in-client`: server-only variable names are forbidden in browser/client paths.
- `provider-key-in-client`: direct provider key variable names are forbidden in client code outside exact classified local-demo/internal-dev debt paths.
- `secret-storage-key`: browser storage writes to API-key, provider-key, token, service-role, or secret-like keys are forbidden.
- `raw-secret-literal`: high-confidence committed secret-looking literals are forbidden.

## Script Behavior Summary

- Uses Node built-ins only.
- Discovers files with `git ls-files --cached --others --exclude-standard`.
- Skips `.git`, `node_modules`, `dist`, `build`, `coverage`, `.turbo`, `.vite`, and `.agent`.

- Scans text files only.
- Allows env variable names in docs, tests, scan scripts, and Supabase Edge/server files when they are names only.

- Prints only rule ID, file path, line number, and classification for forbidden hits.
- Does not print matching line text or matched values.
- Exits non-zero on forbidden hits.
