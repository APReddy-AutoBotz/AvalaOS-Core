## RLS Helper Evidence Requirements

Future AP-approved RLS helper evidence must prove only the approved assertion.

It must define sanitized proof that:

- helper behavior is scoped to the intended tenant, organization, and workspace boundary;
- helper behavior does not allow cross-tenant access;
- helper behavior fails closed on missing or invalid context;
- helper behavior does not infer tenant context from unsafe sources;
- helper behavior produces only sanitized evidence;
- pass and fail criteria are explicit before execution;
- evidence distinguishes helper behavior from full RLS readiness.

It must not capture row payloads, auth headers, claim values, DB URLs, local values, secrets, provider keys, service-role values, private tokens, hosted identifiers, or machine-specific values.
