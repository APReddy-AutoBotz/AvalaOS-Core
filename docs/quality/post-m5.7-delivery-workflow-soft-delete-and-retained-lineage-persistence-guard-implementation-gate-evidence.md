## Implementation Evidence

- Clean delete requests now resolve to source-level `soft_deleted` state with deletion actor/time metadata and no physical delete path.
- Retained-lineage delete requests now resolve to source-level `retained` state with retention class and reason metadata.
- Hard-delete denial semantics remain preserved in `resolveTaskDeletionGuard`.
- Provider delete now saves source state through `deliveryAdapter.saveTask` and records sanitized activity.
- Adapter metadata preserves deletion state, deletion mode, deletion request metadata, deleted metadata, retention reason/class, and restore eligibility through the existing source-level task metadata path.
- Active delivery views filter inactive tasks; Delivery Pack receives full task state for lineage review.
- UI delete copy describes removal from active delivery views, not permanent deletion.
