## Schema/RLS Migration Summary

The migration adds provider governance schema/RLS for:

1. `ai_provider_key_refs`
2. `ai_provider_configs`
3. `ai_workspace_provider_policies`
4. `ai_provider_audit_events`

Final provider list: `gemini`, `groq`.

The migration uses `org_id` as the enforceable tenant boundary. `ai_workspace_provider_policies` does not include `workspace_id` or workspace-based uniqueness. `ai_provider_audit_events.workspace_id` remains nullable without a strict FK because no AP-approved workspace authority exists yet. `ai_provider_key_refs` has no FK back to provider configs. `ai_provider_configs.key_ref_id` is nullable and references `ai_provider_key_refs(id)`.

Repository inspection found no committed `supabase/migrations` directory before hardening. Existing SQL schema references live under `docs/schema`; `docs/schema/ai_governance_jobs.sql` already defines `ai_provider_configs`. The hardening commit uses a Supabase CLI-style migration path and documents the pre-existing table context.

Created when absent:

- `ai_provider_key_refs`
- `ai_workspace_provider_policies`
- `ai_provider_audit_events`

Created when absent or altered when pre-existing:

- `ai_provider_configs`

Pre-existing `ai_provider_configs` notes:

- Prior context already included `org_id`, `provider`, `display_name`, `key_reference`, `default_model`, `model_policy`, `status`, creator/timestamp fields, a member SELECT policy, and supporting indexes/triggers.
- M3.2h adds missing approved governance columns, constraints, indexes, comments, and RLS policy changes.
- M3.2h narrows the provider constraint to `gemini` and `groq`; when a prior table exists, the new provider constraint is added as `NOT VALID` so prior-context rows are not blindly dropped during migration.
- M3.2h does not blindly drop/recreate the table and does not drop prior-context data.
- The prior `key_reference` column is made nullable when present and commented as deprecated; it is not authorized for raw provider keys, encrypted provider keys, auth headers, browser-readable secret values, or secret-manager payload values.

## Final Table Shapes

`ai_provider_key_refs`:

- Columns: `id`, `org_id`, `provider`, `resolver_type`, `secret_ref`, `safe_label`, `safe_fingerprint`, `status`, `rotation_status`, `last_rotated_at`, `expires_at`, `created_by`, `updated_by`, `created_at`, `updated_at`, `deleted_at`.
- Provider values: `gemini`, `groq`.
- Sensitive fields prohibited: raw provider keys, encrypted provider keys, auth headers, browser-readable secret values, and secret-manager payload values.

`ai_provider_configs`:

- Columns: `id`, `org_id`, `provider`, `display_name`, `key_ref_id`, `default_model`, `model_policy`, `allowed_modes`, `allowed_operations`, `evidence_ref`, `status`, `created_by`, `updated_by`, `created_at`, `updated_at`, `deleted_at`.
- Provider values: `gemini`, `groq`.
- Pre-existing `key_reference`, when present, is nullable/deprecated and not authorized for raw secrets.

`ai_workspace_provider_policies`:

- Columns: `id`, `org_id`, `provider_config_id`, `operation`, `mode`, `allowed_roles`, `is_default`, `status`, `created_by`, `updated_by`, `created_at`, `updated_at`, `deleted_at`.
- Constraints/indexes: `operation` not empty; `mode` not empty; status constrained to `pending_review`, `active`, `disabled`, `retired`; one active default per `(org_id, operation, mode)` where `is_default = true` and `deleted_at IS NULL`; active non-deleted uniqueness for `(org_id, provider_config_id, operation, mode)`.
- Excluded from final shape: `workspace_id`, `policy_name`, `allowed_modes`, `allowed_operations`, `max_prompt_tokens`, `max_output_tokens`, `policy_document`, `evidence_ref`, `effective_from`, `effective_until`.

`ai_provider_audit_events`:

- Columns: `id`, `schema_version`, `event_type`, `org_id`, `workspace_id`, `provider`, `provider_config_id`, `key_ref_id`, `operation`, `mode`, `policy_result`, `status`, `failure_class`, `actor_id`, `service_context`, `correlation_id`, `evidence_ref`, `metadata`, `created_at`.
- Provider values: `gemini`, `groq`.
- `workspace_id` is nullable without FK and commented as deferred until AP approves a workspace authority.
- `metadata` is JSONB and blocks common raw secret, auth-header, raw prompt, and raw completion field names.

## Secret-Safety Confirmation

- `secret_ref` is documented as a service-only reference identifier.
- `secret_ref` must not contain raw provider keys, encrypted provider keys, auth headers, browser-readable secret values, or secret-manager payload values.
- `ai_provider_configs` is documented as prohibited from storing raw secrets, encrypted secrets, auth headers, browser-readable secret values, or secret-manager payload values.
- Browser/client code does not resolve provider keys in M3.2h.
- Browser/client code does not directly write sensitive provider/key/audit objects in M3.2h.

- `ai_provider_configs`: RLS enabled; active org members can SELECT non-deleted metadata; no browser/client write policy.
- `ai_workspace_provider_policies`: RLS enabled; active org members can SELECT non-deleted metadata; no browser/client write policy.
